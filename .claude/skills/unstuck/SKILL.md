# 卡文救援

分析当前剧情状态，提供多个可能的推进方向。

## 执行流程

### 1. 加载当前状态
- `06_追踪/伏笔追踪.md`（未回收的伏笔）
- `03_大纲/` 当前卷细纲/大纲
- `04_章纲/` 最近完成的章纲
- `05_正文/` 最近一章的结尾

### 2. 分析瓶颈
向作者确认卡在哪里：
- **情节卡点**：不知道接下来该发生什么
- **人物卡点**：角色行为不合理，无法推进
- **结构卡点**：节奏失控，不知道如何收束
- **设定卡点**：世界观细节不足，无法展开

### 3. 生成方向
根据卡点类型，提供 3-5 个不同的推进方向：

**情节卡点**：
- 检索未回收伏笔，找出可以激活的线索
- 从关系网中找可以介入的角色
- 引入意外变量（第三方势力/新信息/突发事件）

**人物卡点**：
- 回溯角色动机，找到被忽略的利益驱动
- 考虑角色的"非理性选择"——矛盾情感导致的意外行为
- 让其他角色的行动迫使当前角色做出反应

**结构卡点**：
- 检查十大序列位置，确认当前应处于哪个节奏段
- 考虑是否需要插入过渡场景或视角切换
- 评估是否需要加速或减速

**设定卡点**：
- 检查世界观 INDEX，找出未展开的设定线索
- 通过角色对话自然引入新设定
- 用"限制"替代"扩展"——给已有设定加新约束

### 4. 输出
每个方向包含：
- 一句话概述
- 涉及的角色和场景
- 对后续情节的影响
- 潜在风险

**等待作者选择。**
