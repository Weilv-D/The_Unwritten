# 每日创作启动

为今日创作做准备，自动加载所需上下文并展示项目状态。

## 执行步骤

### 1. 读取项目状态
- 读取 `00_项目基础/进度追踪.md`
- 读取 `00_项目基础/任务清单.md`
- 读取 `06_追踪/创作日志.md`（最后3条记录）

### 2. 统计字数
扫描 `05_正文/` 目录下所有文件，计算总字数（中文字符数）。按卷次分别统计。

### 3. 判断当前阶段
根据仓库实际文件状态判断当前阶段，忽略 `INDEX.md`、模板文件与 `.gitkeep`：
- 若 `05_正文/卷X/chNNN.md` 已存在正文文件，当前阶段视为“正文”
- 若只有 `04_章纲/卷X/chNNN.md`，当前阶段视为“章纲”
- 若已有 `03_大纲/卷次/卷X_大纲.md` 或 `卷X_细纲.md`，当前阶段视为“大纲”
- 若已有作品级角色档案，当前阶段视为“人物”
- 否则视为“世界观/工作台待绑定作品”

### 4. 汇报当前状态

向作者展示：

```
## 项目状态
### 整体进度
- 总字数：XXXX
- 目标字数：[未锁定 / XXXX]
- 进度：[目标未锁定时不显示百分比；已锁定时显示 X.X%]
- 当前阶段：[世界观/人物/大纲/章纲/正文]
- 当前卷次：卷X

### 各卷进度
| 卷次 | 字数 | 章节 | 状态 |
|------|------|------|------|
| 卷1 | XXXX | X/已锁定章节数或未锁定 | 进行中 |

### 最近活动
- [最近3条创作日志]

### 待办事项
- [从任务清单中提取高优先级项]
```

如果 `00_项目基础/进度追踪.md` 中的目标字数为“待定”或未锁定，明确写成“未锁定”，不要自行补出百分比。

### 5. 确认今日目标

**必须询问作者**：
- 今天要做什么？（写作/审阅/设定/规划/灵感）
- 目标章节或任务
- 预计时长

### 6. 预加载上下文
根据今日目标自动加载对应素材，但只读取**实际存在**的文件：
- 写作：章纲 + 人物速查 + 上章结尾（存在时）
- 审阅：待审正文 + 对应章纲
- 设定：相关现有设定文件
- 规划：大纲 + 已有设定
- 灵感：`08_灵感/INDEX.md`

如预期文件不存在，直接在状态汇报中标注缺口，不引用不存在的路径。

### 7. 就绪确认
告知作者已加载的文件列表，准备开始工作。

### 8. 建议
根据当前进度，建议下一步操作。
