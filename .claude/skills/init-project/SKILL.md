# 初始化项目

幂等检查并创建工作区所需的所有基础设施文件。

## 执行流程

### 1. 扫描缺失文件

检查以下文件是否存在，列出缺失项：

**核心基础设施：**
- `02_人物/人物速查.md`
- `02_人物/INDEX.md`
- `01_世界观/INDEX.md`
- `03_大纲/INDEX.md`
- `03_大纲/故事梗概.md`
- `06_追踪/创作日志.md`
- `06_追踪/伏笔追踪.md`
- `00_项目基础/进度追踪.md`
- `00_项目基础/任务清单.md`

**灵感目录：**
- `08_灵感/INDEX.md`
- `08_灵感/剧情灵感.md`
- `08_灵感/场景灵感.md`
- `08_灵感/对话碎片.md`
- `08_灵感/主题思考.md`
- `08_灵感/技法笔记.md`
- `08_灵感/待分类.md`

**空目录 .gitkeep：**
- `08_灵感/研究资料/.gitkeep`
- `00_项目基础/.gitkeep`（仅当目录完全为空时）

### 2. 报告

向作者展示：
- 已存在的文件数量
- 缺失文件列表
- 建议操作

### 3. 确认

**等待作者确认后**再创建文件。

### 4. 创建缺失文件

为每个缺失文件创建种子内容：

**人物速查**：空表格（角色/定位/描述/状态/路径）
**INDEX 文件**：空表格，状态列标"待设计"
**故事梗概**：待填充占位（类型/规模/主题/主角）
**创作日志**：提示"日志将在首次操作后自动生成"
**伏笔追踪**：三张空表（活跃/已回收/跨卷长线）
**进度追踪**：5卷空表格 + 里程碑列表
**任务清单**：3项初始任务（确定信息/世界观/主角）
**灵感文件**：标题 + "暂无记录"

### 5. 验证

创建完成后：
- 列出所有已创建文件
- 确认目录结构完整
- 建议提交 Git
