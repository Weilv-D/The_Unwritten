# 编辑章节

## 必读

- `.claude/rules/core-style.md`
- `04_章纲/$ARGUMENTS.md`
- 相关审查报告或作者反馈。

## 流程

1. **加载上下文清单**
   - 当前正文：`05_正文/$ARGUMENTS.md`
   - 对应章纲与写作契约。
   - 审查报告、作者反馈、相关设定源文件。
   - 缺正文或章纲时停止。

2. **确认编辑范围**
   - 明确修改类型：微调、段落级、场景级、结构性。
   - 明确不可改动段落和必须保留的章纲功能。
   - 如修改会改变剧情、人物命运、canon 设定或伏笔回收，先请作者决策。

3. **证据优先修订**
   - 每处修改对应一个问题或反馈，不做无目标润色。
   - 保持写作契约不变；如契约需要变更，先回到 `/design-chapter-outline`。
   - 涉及术语或能力边界时回源文件查证。

4. **反思修订**
   - 修后列出 3-5 个残余风险：连续性、角色动机、术语、节奏、结尾牵引。
   - 必要时再做一次定向修订。

5. **FAIL级硬规则自检**
   - 逐条标注 PASS/FAIL。
   - FAIL 项必须给出位置、触发规则和进一步修改建议。

6. **输出与存档**
   - 先展示修改位置、修改原因和自检结果。
   - 作者确认后存档，覆盖 `05_正文/$ARGUMENTS.md` 并更新 `06_追踪/创作日志.md`。
