# 会话收尾

总结本次会话成果，为下次会话准备上下文。

## 执行步骤

### 1. 回顾本次会话
优先结合以下信息回顾本次会话：
- 当前仓库的 `git status --short`
- 当前未提交的变更文件列表
- `06_追踪/创作日志.md` 最近记录

`创作日志.md` 仅作辅助日志，不应被视为本次会话的唯一事实来源。

### 2. 总结成果
向作者展示：
- 本次完成的工作（写了哪些章节/修改了哪些设定/完成了哪些规划）
- 关键决策记录（做了什么创作选择）
- 字数变化（如有正文创作）

### 3. 确认待办

**询问作者**：
- 有没有需要记录的待办事项？
- 有没有延迟到下次的决定？
- 有没有本次发现的问题需要下次处理？

### 4. 更新进度
- 更新 `00_项目基础/进度追踪.md`
- 更新 `00_项目基础/任务清单.md`（如有新增待办）
- 更新 `06_追踪/伏笔追踪.md`（如有伏笔状态变化）
- 更新 `06_追踪/知识图谱.md`（如有关系网变化，建议运行 `/kg-update`）
- 检查 Git 工作区状态；如仍有未提交变更，提示作者在确认后统一提交

### 5. 建议下次目标
根据当前进度和待办事项，建议下次会话的目标。
