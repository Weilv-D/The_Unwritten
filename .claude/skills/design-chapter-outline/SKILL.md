# 设计章纲

## 必读

- `.claude/rules/core-plot.md`
- `.claude/rules/core-style.md`
- `04_章纲/章纲模板.md`

## 输入

章节编号：`$ARGUMENTS`，格式如 `卷1/ch003`。

## 流程

1. **加载上下文清单**
   - 从 `03_大纲/卷次/<卷次>_细纲.md` 提取本章条目。
   - 读取前一章章纲（如有）、相关人物档案、涉及的世界观源文件。
   - 如果细纲不存在，停止并建议先运行 `/design-detail-outline <卷次>`。

2. **确认本章设计**
   - 向作者展示序列位置、叙事功能、POV、场景、基调、伏笔需求。
   - 如涉及未锁定的作品实体或 canon 事实，等待作者确认，不自行补完。

3. **生成章纲**
   - 使用 `章纲模板.md`。
   - 必须包含“写作契约”，列出 POV 边界、场景目标、阻碍、意外变量、决断转折、信息释放、禁止事项、需查证来源。
   - 必须包含“来源清单”“不确定项”“伏笔操作”。
   - 三幕比例是默认模板，可按章功能偏离，并在元信息中说明。

4. **自检**
   - 场景公式是否完整：POV 目标 + 阻碍/信息差 + 意外变量 + 决断转折。
   - 是否存在 POV 越界、说明书式设定倾倒、未查证术语、未确认 canon。
   - 若发现冲突，标入“不确定项”，不自行消解。

5. **写入**
   - 章纲写入 `04_章纲/$ARGUMENTS.md`。
   - 提醒作者审核后再进入 `/write-chapter`。

## 错误处理

- **细纲缺失或不足**：停止，建议先完善细纲。
- **与前章冲突**：标出冲突位置，请作者决断。
- **术语未查证**：列出需查证源文件，不凭 `INDEX.md` 释义扩写。
