# 设计章纲

从细纲生成单章写作蓝图，写作前的最后规划。

## 必读规则
- `core-plot.md`（微观场景引擎、场景推演公式）
- `core-style.md`（视角纪律、术语规范）

## 输入参数
章节编号：$ARGUMENTS（格式：卷1/ch003）

## 执行步骤

### 1. 加载上下文
- `03_大纲/$ARGUMENTS_细纲.md` 中对应本章的条目
- `04_章纲/章纲模板.md`
- 前一章章纲（如有）
- 视角人物档案
- 场景涉及的世界观设定

如果细纲不存在，建议先运行 `/design-detail-outline`。

### 2. 确认本章设计

向作者展示并确认：
- 本章在十大序列中的位置
- 核心叙事功能（推动情节/揭示信息/展现代价/深化主题）
- 视角人物和场景
- 三幕结构要点（钩子→博弈→收束）
- 预计基调（肃杀/宏大/幽静/诙谐/阴沉）
- 需要注意的伏笔或连贯性

**等待作者确认。**

### 3. 生成章纲

按章纲模板格式输出，包含：

**元数据**：
- 卷次/序号/视角人物/字数目标/基调

**三幕结构**：
```
第一幕：钩子切入（约1500字）
  - 开场镜头（环境渲染+悬念）
  - 核心事件触发
  - 钩子句

第二幕：发展博弈（约4500字）
  - 对话/行动/视角切换
  - 信息三层释放
  - 人物状态变化
  - 场景过渡点

第三幕：收束留白（约1500字）
  - 阶段性结果
  - 新伏笔埋设
  - 尾声镜头
```

**八大要素填充**：
- 身份/欲望/动作/问题/阻力/结果/正负主题

**人物出场清单**：
- 每人本章目标、态度变化、关键台词方向

**连贯性钩点**：
- 承接上章的线索
- 需要设置/回收的伏笔

### 4. 输出
章纲写入 `04_章纲/$ARGUMENTS.md`，请作者审核确认。

## 错误处理
- **细纲条目不足**：停止并建议先完善细纲
- **与前章章纲冲突**：标记冲突，请作者决断
- **伏笔盲点**：停止并报告，不自行消解
