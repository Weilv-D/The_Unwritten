# 写作正文

按完整流程创作正文。支持**章节正文**与**短篇/练习**两种模式，根据输入参数自动路由。

## 必读规则
- `core-style.md`（硬规则、默认模板、文风锚定）
- `core-character.md`（角色自主性、情感克制）

## 输入参数

$ARGUMENTS 决定模式：

| 输入格式 | 模式 | 示例 |
|---------|------|------|
| `卷X/chNNN` | 章节正文 | `卷1/ch003` |
| `短篇/标题` | 短篇/练习 | `短篇/河洛雨` |
| 其他文本 | 视作需求描述，进入交互确认 | `用许遂写一篇角木道途的短篇` |

## 上下文预算

| 优先级 | 内容 | 模式 | 预估 tokens |
|--------|------|------|-------------|
| 必读 | `core-style.md` | 通用 | ~300 |
| 必读 | `core-character.md` | 通用 | ~400 |
| 必读 | 章纲 `04_章纲/卷X/chNNN.md` | 仅章节 | ~400 |
| 必读 | 人物速查 `02_人物/人物速查.md` | 通用 | ~500 |
| 必读 | 风格锚定 `00_项目基础/风格锚定.md`（如存在） | 通用 | ~300 |
| 通常 | 上章结尾 `05_正文/卷X/chNNN.md` | 仅章节 | ~800 |
| 通常 | 世界观摘要 `01_世界观/INDEX.md` | 通用 | ~500 |
| 按需 | 伏笔追踪、特定人物档案 | 通用 | ~500 |

每读一个文件前评估必要性，不读取无关文件。

## 执行步骤

### 0. 模式判断

根据 $ARGUMENTS 判断模式：

- **格式 `卷X/chNNN`** → 章节模式（走步骤 1A→2A→3→4→5→6）
- **格式 `短篇/标题`** → 短篇模式（走步骤 1B→2B→3→4→5→6）
- **其他** → 进入交互确认，由作者指定模式

---

### 1A. 章节模式：加载上下文

**必读**：
- `04_章纲/卷X/chNNN.md` 当前章纲
- `02_人物/人物速查.md` 视角人物速览
- `00_项目基础/风格锚定.md`（如存在）

**通常需要**：
- `05_正文/卷X/` 中前一章文件（用于衔接）
- `01_世界观/INDEX.md` 当前场景相关设定

**按需**：伏笔追踪表、其他人物档案

如果章纲文件不存在，告知作者并建议先运行 `/design-chapter-outline`，不自行跳过章纲。

### 2A. 章节模式：连续性预检

如果前一章正文存在，提取并向作者展示：
- **上章结尾场景**：时间、地点、在场人物
- **情绪状态**：视角人物和关键角色的最后情绪
- **未解决张力**：悬念、冲突、未回应的对话
- **本章必须回应的钩子**：上章结尾留下的任何悬念或张力

向作者确认连续性无误。

---

### 1B. 短篇模式：确认需求

**必读**：
- 涉及人物对应的角色档案（速览段 + 欲望/弱点/弧光）
- `00_项目基础/风格锚定.md`（如存在）

**建议**（向作者展示选项，等待确认）：
- 世界观摘要 `01_世界观/INDEX.md`（如涉及特定势力/地域）
- 其他角色档案（如涉及多人互动）

**与作者确认**：
- 核心主题或叙事目标（探索角色/练习文风/测试设定等）
- 使用哪些人物
- 预计篇幅（短篇默认 2000-4000 字，按需调整）
- 是否需要与现有设定挂钩，还是独立成篇

### 2B. 短篇模式：创作预检

**检查项**：
- 所选人物的核心欲望与当前状态是否与档案一致
- 是否存在尚未建档但需要使用的角色（如有，先运行 `/create-character`）
- 短篇中的情节/对话是否与现有设定产生冲突

向作者确认创作方向无误。

---

### 3. 确认创作方向

向作者展示本章/本篇关键信息，**等待确认**：
- 核心叙事功能（推动情节/揭示信息/展现代价/深化主题/练习目的）
- 视角人物和场景
- 结构要点（钩子→博弈→收束）
- 预计基调（肃杀/宏大/幽静/诙谐/阴沉）
- 需要注意的伏笔或连贯性

### 4. 创作正文

严格遵循风格规范：
- **字数**：章节默认 7500 字（6500-8500），短篇默认 2500-4000 字；偏离时说明原因
- **格式**：纯文本，不输出md标记
- **风格**：半文半白；严格遵循 `core-style.md` 的视角纪律和展示优先原则
- **内心独白**：按需使用 `「」`，不要求高频出现
- **信息密度**：每段都有信息增量，不为凑篇幅注水
- **风格锚定**：如存在 `00_项目基础/风格锚定.md`，参照黄金段落保持风格一致

### 5. 自检

创作完成后，执行**首稿快速自检**：
- 连续性是否成立（承接、人物状态、时间地点）
- `core-style.md` 中的 FAIL 级硬规则是否触发
- 是否存在明显术语冲突、POV 越界、现代口语、说明书式设定倾倒
- 短篇模式下额外检查：是否与现有设定/人物档案产生冲突

如作者要求终稿级精修，再追加 `core-style.md` 中的深审项与文言细查。

### 6. 输出
- 正文内容
- 自检报告
- 询问作者是否需要修改

### 7. 存档
作者确认后：
- **章节**：正文写入 `05_正文/卷X/chNNN.md`；更新 `06_追踪/创作日志.md` 与 `06_追踪/伏笔追踪.md`
- **短篇**：正文写入 `08_灵感/短篇故事_标题.md`；更新 `08_灵感/INDEX.md`
- 建议提交 Git

## 错误处理
- **字数偏离默认范围**：说明原因，不为凑字数而注水
- **发现设定盲点**：停止并向作者报告，不自行消解
- **章节模式无法衔接上章**：标记问题点，请作者确认
- **短篇模式角色未建档**：建议先运行 `/create-character`
