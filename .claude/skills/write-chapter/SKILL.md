# 写作正文

## 必读

- `.claude/rules/core-style.md`
- `.claude/rules/core-character.md`
- `00_项目基础/风格锚定.md`

## 输入模式

| 输入格式 | 模式 |
|----------|------|
| `卷X/chNNN` | 章节正文 |
| `短篇/标题` | 短篇/练习/文风试笔 |
| 其他文本 | 需求描述，先请作者确认模式 |

## 执行顺序

**上下文清单 → 连续性/设定预检 → 草稿 → 反思修订 → FAIL级硬规则自检 → 作者确认后存档**

## 1. 上下文清单

按需读取，避免一次性加载无关世界观。

章节模式必读：
- `04_章纲/$ARGUMENTS.md`，必须含“写作契约”。
- `02_人物/人物速查.md` 与 POV 人物档案。
- `00_项目基础/风格锚定.md`。

章节模式通常需要：
- 前一章正文与前一章章纲（如存在）。
- 章纲“来源清单”列出的世界观源文件。
- `06_追踪/伏笔追踪.md` 中与本章相关条目。

短篇模式必读：
- 涉及人物档案。
- `00_项目基础/风格锚定.md`。
- 涉及术语或势力时读取对应世界观源文件。

如果章节模式缺章纲，停止并建议先运行 `/design-chapter-outline $ARGUMENTS`。

## 2. 连续性/设定预检

章节模式先输出并请作者确认：
- 上章结尾：时间、地点、在场人物、未解决张力。
- 本章写作契约：POV、场景目标、阻碍、意外变量、决断转折、信息释放。
- 需查证来源：术语、势力、功法、人物状态对应源文件。

短篇模式先输出并请作者确认：
- 叙事目标、人物、篇幅、基调。
- canon 状态：`正式采纳 / 试笔参考 / 不入库`。
- 是否允许把新设定写入正式资料。默认不允许。

发现设定盲点、角色未建档、来源缺失或关键冲突时，停止并报告。

## 3. 草稿

- 章节默认 7500 字，6500-8500 为常用参考区间；短篇默认 2500-4000 字。
- 纯正文输出，不用 Markdown 标题包装正文。
- 遵守写作契约和风格锚定：半文半白、限知 POV、展示优先、对话即博弈。
- 复杂设定通过行动、对话、物件和利益分配侧面呈现，不做说明书式倾倒。

## 4. 反思修订

草稿完成后先列 3-5 个高风险点，再执行一次定向修订：
- 连续性是否断裂。
- 角色选择是否违背欲望、弱点、位置。
- 术语是否未查证或误用。
- 对话是否只在解释设定。
- 结尾是否缺少留白或下章牵引。

修订只处理实际问题，不为凑字数或追求华丽而堆词。

## 5. FAIL级硬规则自检

逐条标注 PASS/FAIL：
- 术语与设定一致性。
- POV 限知。
- 无说明书式设定倾倒。
- 角色或势力不为推剧情降智。
- 无明显现代口语、网络语、现代社科套话。
- 关键冲突未被私自消解。

如作者要求终稿级精修，再追加 `core-style.md` 深审项：实词、虚词、名词、可读性。

## 6. 输出与存档

先输出正文、反思修订说明和自检报告。

作者确认后存档：
- 章节：写入 `05_正文/卷X/chNNN.md`，再更新 `06_追踪/创作日志.md` 与必要的伏笔追踪。
- 短篇：仅当作者确认存档时写入 `08_灵感/短篇故事_标题.md` 并更新 `08_灵感/INDEX.md`。
- canon 状态为 `试笔参考` 或 `不入库` 时，不得自动同步世界观、人物档案或伏笔事实。
