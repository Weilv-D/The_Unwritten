# 写作章节

按完整流程创作一章《坠玉京》正文。

## 输入参数
章节编号：$ARGUMENTS（格式：卷1/ch003）

## 执行流程

### 1. 加载上下文（按优先级）
**必读**（~2000字）：
- `CLAUDE.md` 风格速查部分
- `04_章纲/$ARGUMENTS.md` 当前章纲
- `02_人物/人物速查.md` 视角人物速览

**通常需要**（~2000字）：
- `05_正文/` 中前一章文件（用于衔接）
- `01_世界观/INDEX.md` 当前场景相关设定

**按需**：伏笔追踪表、其他人物档案

如果章纲文件不存在，告知作者并建议先运行 `/design-chapter-outline`。

### 2. 确认创作方向
向作者展示本章关键信息，**等待确认**：
- 核心叙事功能（推动情节/揭示信息/展现代价/深化主题）
- 视角人物和场景
- 三幕结构要点（钩子→博弈→收束）
- 预计基调（肃杀/宏大/幽静/诡谲/阴沉）
- 需要注意的伏笔或连贯性

### 3. 创作正文
严格遵循风格规范：
- **字数**：6000字（严格要求）
- **格式**：纯文本，不输出md标记
- **风格**：半文半白，[景-人-心]段落公式
- **内心独白**：高频使用『』，表里反差
- **信息密度**：每段都有信息增量

### 4. 自检
创作完成后，执行 12 条禁止规则逐条检查：
- 每条标注 PASS/FAIL
- FAIL 项标注具体位置和修改建议

### 5. 输出
- 正文内容
- 自检报告
- 询问作者是否需要修改

### 6. 存档
作者确认后：
- 正文写入 `05_正文/$ARGUMENTS.md`
- 更新 `06_追踪/创作日志.md`
- 更新 `06_追踪/伏笔追踪.md`（如有新增或回收）
- 建议提交 Git
