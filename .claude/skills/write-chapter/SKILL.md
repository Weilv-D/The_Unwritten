# 写作章节

按完整流程创作一章正文。

## 输入参数
章节编号：$ARGUMENTS（格式：卷1/ch003）

## 上下文预算

**总预算约 20,000 tokens，规则层已占约 12,000，内容层可用约 8,000。**

| 优先级 | 内容 | 预估 tokens |
|--------|------|-------------|
| 必读 | 章纲 `04_章纲/$ARGUMENTS.md` | ~400 |
| 必读 | 人物速查 `02_人物/人物速查.md` | ~500 |
| 必读 | 风格锚定 `00_项目基础/风格锚定.md`（如存在） | ~300 |
| 通常 | 上章结尾 `05_正文/` 前一章 | ~800 |
| 通常 | 世界观摘要 `01_世界观/INDEX.md` | ~500 |
| 按需 | 伏笔追踪、特定人物档案 | ~500 |

每读一个文件前评估必要性，不读取无关文件。

## 执行流程

### 1. 加载上下文（按优先级）
**必读规则**：`writing-style.md`、`forbidden-rules.md`、`creative-philosophy.md`、`methodology-writing.md`

**必读**：
- `04_章纲/$ARGUMENTS.md` 当前章纲
- `02_人物/人物速查.md` 视角人物速览
- `00_项目基础/风格锚定.md`（如存在）

**通常需要**：
- `05_正文/` 中前一章文件（用于衔接）
- `01_世界观/INDEX.md` 当前场景相关设定

**按需**：伏笔追踪表、其他人物档案

如果章纲文件不存在，告知作者并建议先运行 `/design-chapter-outline`。

### 2. 连续性预检
如果前一章正文存在，提取并向作者展示：
- **上章结尾场景**：时间、地点、在场人物
- **情绪状态**：视角人物和关键角色的最后情绪
- **未解决张力**：悬念、冲突、未回应的对话
- **本章必须回应的钩子**：上章结尾留下的任何铺垫或悬念

向作者确认连续性无误。

### 3. 确认创作方向
向作者展示本章关键信息，**等待确认**：
- 核心叙事功能（推动情节/揭示信息/展现代价/深化主题）
- 视角人物和场景
- 三幕结构要点（钩子→博弈→收束）
- 预计基调（肃杀/宏大/幽静/诡谲/阴沉）
- 需要注意的伏笔或连贯性

### 4. 创作正文
严格遵循风格规范：
- **字数**：6000字（最低5500，软上限6500）
- **格式**：纯文本，不输出md标记
- **风格**：半文半白，[景-人-心]段落公式
- **内心独白**：高频使用『』，表里反差
- **信息密度**：每段都有信息增量
- **风格锚定**：如存在 `00_项目基础/风格锚定.md`，参照黄金段落保持风格一致

### 5. 自检
创作完成后，执行 12 条禁止规则逐条检查：
- 每条标注 PASS/FAIL
- FAIL 项标注具体位置和修改建议

### 6. 输出
- 正文内容
- 自检报告
- 询问作者是否需要修改

### 7. 存档
作者确认后：
- 正文写入 `05_正文/$ARGUMENTS.md`
- 更新 `06_追踪/创作日志.md`
- 更新 `06_追踪/伏笔追踪.md`（如有新增或回收）
- 建议提交 Git

## 错误处理
- **字数不足 5500**：停止并说明原因，不凑字数
- **发现设定矛盾**：停止并向作者报告，不自行消解
- **无法衔接上章**：标记问题点，请作者确认
