# 写作章节

按完整流程创作一章正文。

## 必读规则
- `core-style.md`（硬规则、默认模板、文风锚定）
- `core-character.md`（角色自主性、情感克制）

## 输入参数
章节编号：$ARGUMENTS（格式：卷1/ch003）

## 上下文预算

| 优先级 | 内容 | 预估 tokens |
|--------|------|-------------|
| 必读 | 章纲 `04_章纲/$ARGUMENTS.md` | ~400 |
| 必读 | 人物速查 `02_人物/人物速查.md` | ~500 |
| 必读 | 风格锚定 `00_项目基础/风格锚定.md`（如存在） | ~300 |
| 通常 | 上章结尾 `05_正文/` 前一章 | ~800 |
| 通常 | 世界观摘要 `01_世界观/INDEX.md` | ~500 |
| 按需 | 伏笔追踪、特定人物档案 | ~500 |

每读一个文件前评估必要性，不读取无关文件。

## 执行步骤

### 1. 加载上下文（按优先级）

**必读**：
- `04_章纲/$ARGUMENTS.md` 当前章纲
- `02_人物/人物速查.md` 视角人物速览
- `00_项目基础/风格锚定.md`（如存在）

**通常需要**：
- `05_正文/` 中前一章文件（用于衔接）
- `01_世界观/INDEX.md` 当前场景相关设定

**按需**：伏笔追踪表、其他人物档案

如果章纲文件不存在，告知作者并建议先运行 `/design-chapter-outline`。

### 2. 连续性预检

如果前一章正文存在，提取并向作者展示：
- **上章结尾场景**：时间、地点、在场人物
- **情绪状态**：视角人物和关键角色的最后情绪
- **未解决张力**：悬念、冲突、未回应的对话
- **本章必须回应的钩子**：上章结尾留下的任何悬念或张力

向作者确认连续性无误。

### 3. 确认创作方向

向作者展示本章关键信息，**等待确认**：
- 核心叙事功能（推动情节/揭示信息/展现代价/深化主题）
- 视角人物和场景
- 三幕结构要点（钩子→博弈→收束）
- 预计基调（肃杀/宏大/幽静/诙谐/阴沉）
- 需要注意的伏笔或连贯性

### 4. 创作正文

严格遵循风格规范：
- **字数**：默认目标 7500 字；若低于 6500 或高于 8500，应说明原因，不把字数本身视为失败条件
- **格式**：纯文本，不输出md标记
- **风格**：半文半白；`[景/人/心]` 可作为常用组织方式，但不是逐段硬模板
- **内心独白**：按需使用 `「」`，不要求高频出现
- **信息密度**：每段都有信息增量
- **风格锚定**：如存在 `00_项目基础/风格锚定.md`，参照黄金段落保持风格一致

### 5. 自检

创作完成后，先执行**首稿快速自检**：
- 连续性是否成立（承接、人物状态、时间地点）
- `core-style.md` 中的 FAIL 级硬规则是否触发
- 是否存在明显术语冲突、POV 越界、现代口语、说明书式设定倾倒

如作者要求终稿级精修，再追加 `core-style.md` 中的深审项与文言细查。

### 6. 输出
- 正文内容
- 自检报告
- 询问作者是否需要修改

### 7. 存档
作者确认后：
- 正文写入 `05_正文/$ARGUMENTS.md`
- 更新 `06_追踪/创作日志.md`
- 更新 `06_追踪/伏笔追踪.md`（如有新增或回收）
- 建议提交 Git

## 错误处理
- **字数偏离默认范围**：说明原因，不为凑字数而注水
- **发现设定盲点**：停止并向作者报告，不自行消解
- **无法衔接上章**：标记问题点，请作者确认
