# 一致性检查

对指定范围进行系统性的跨文件一致性检查。

## 必读规则
- `core-character.md`
- `core-world.md`

## 执行步骤

### 1. 确定范围
- 单章检查：指定章节
- 全卷检查：指定卷次
- 如用户未指定，询问检查范围

### 2. 五维一致性检查

**A. 角色一致性**
- 语音风格是否前后统一
- 行为选择是否符合性格设定
- 能力使用是否在设定范围内
- 称呼是否一致
- 外貌描写是否前后矛盾

**B. 世界观一致性**
- 地理位置描述是否一致
- 力量体系使用是否符合设定
- 修行体系相关描述是否准确
- 文化习俗是否前后统一
- 关键术语使用是否与源文件定义一致（非仅与 INDEX.md 简称一致）
- 如发现术语可能被误用，必须对比 `01_世界观/` 对应源文件验证

**C. 时间线一致性**
- 事件顺序是否合理
- 季节/天气是否吻合
- 人物位置是否可达
- 信息传递是否超出通讯手段

**D. 伏笔一致性**
- 新埋设的伏笔是否已记录
- 已回收的伏笔是否标记
- 长线伏笔的推进状态

**E. 物品/细节一致性**
- 物品是否凭空出现/消失
- 角色是否知道他不该知道的事
- 法术/功法恢复是否有交代

### 3. 输出报告

```
## 一致性检查报告
**检查范围**：[章节/卷次]

### 角色一致性
| 检查项 | 状态 | 说明 |
|--------|------|------|
| 语音风格 | ✅/⚠️/❌ | 具体说明 |

[其他维度同上格式]

**综合评分**：X/10
**关键问题**：[列出必须修复的问题]
```

### 4. 更新追踪
如发现新的伏笔问题，更新 `06_追踪/伏笔追踪.md`。
