# 一致性检查

## 必读

- `.claude/rules/core-character.md`
- `.claude/rules/core-world.md`
- `06_追踪/审查报告/审查报告模板.md`

## 流程

1. **确定范围**
   - 单章：读取正文、章纲、相关人物与来源清单。
   - 全卷：读取卷大纲、细纲、章纲、正文、伏笔追踪。
   - 资料范围：读取用户指定目录或文件。
   - 范围不明时先问作者，不扩大到全库。

2. **来源取证**
   - 遇到术语、势力、境界、道途、功法，必须读取 `01_世界观/` 对应源文件。
   - `INDEX.md` 只用于定位，不作为完整规则来源。
   - 未查证内容标注“待查证”。

3. **五维一致性**
   - 角色一致性：语音、行为、能力、称呼、外貌。
   - 世界观一致性：地理、力量体系、修行规则、文化习俗、术语。
   - 时间线一致性：事件顺序、季节天气、移动距离、信息传递。
   - 伏笔一致性：新设、推进、回收、遗漏登记。
   - 物品/细节一致性：物品出现消失、法术恢复、角色知情范围。

4. **证据优先报告**
   - 必须修复项在前，建议优化在后。
   - 每项包含：问题位置、触发规则、证据、影响、建议处理、需作者决策项。
   - 综合评分只放末尾。

5. **追踪更新**
   - 先列“建议更新的追踪项”。
   - 仅在作者确认后，才更新 `06_追踪/伏笔追踪.md` 或知识图谱。
