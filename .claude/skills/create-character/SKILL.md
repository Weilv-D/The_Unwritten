# 创建角色

按统一方法论的角色六步法，系统化创建角色档案。

## 必读规则
- `core-character.md`（五维法、自主性三铁律、讨喜范式）

## 输入参数
角色名：$ARGUMENTS（如 主角_姓名）

## 执行步骤

### 1. 收集基础信息
向作者询问（如未提供）：
- 角色在故事中的定位（主角/核心配角/重要配角/功能性角色）
- 一句话定义（这个角色是谁）
- 首次出场位置
- 已有的关系信息

### 2. 提供设计方案
基于 `core-character.md` 的讨喜范式和功能模板，提供 2-3 个设计方案，每个方案包含：
- 核心功能定位
- 讨喜范式选择
- 核心欲望与矛盾
- 标志性细节建议

**等待作者选择或提出修改。**

### 3. 填写完整档案
按 `02_人物/角色模板.md` 格式填写完整角色档案。

### 4. 一致性检查
检查新角色与已有角色的：
- 关系网是否自洽
- 势力归属是否与世界观匹配
- 功能定位是否与已有角色重叠

### 5. 写入文件
- 角色档案：`02_人物/$ARGUMENTS.md`
- 更新人物速查：`02_人物/人物速查.md`
- 更新索引：`02_人物/INDEX.md`

### 6. 确认
向作者展示最终档案，确认是否需要调整。
