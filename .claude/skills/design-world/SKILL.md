# 世界观设计

按涟漪展开法设计和扩展世界观。

## 输入参数
设计目标：$ARGUMENTS（如：地理、势力、经济、文化、历史、种族）

缺省时列出所有待完成模块，请作者选择。

## 执行流程

### 1. 确认设计范围
展示 `01_世界观/INDEX.md` 中该模块的当前状态：
- 已有哪些设定
- 待设计哪些条目
- 与其他模块的依赖关系

### 2. 涟漪层级定位
确认本次设计属于哪一环：
- **第一环**（个人）：修行资源、功法、灵物
- **第二环**（宗门/家族）：灵脉、领地、传承、弟子
- **第三环**（地区格局）：势力范围、贸易路线、秘境
- **第四环**（天下大势）：王朝更替、道统兴衰

### 3. 提供设计方案
基于已有设定，提供 **2-3 个设计方案**，每个包含：
- 核心概念（一句话定义）
- 与力量体系的关联
- 与已知势力的互动
- 可产生的叙事张力点
- 3-5 个关键设定条目

**必须等待作者选定方案。**

### 4. 渐进式展开
对选定方案深入填充：
- 只设计"故事需要显露的 10%"
- 为每个条目标注"对读者可见"还是"仅作者知晓"
- 标注后续可展开方向（90%冰山水下部分）

### 5. 一致性检查
- 与 `01_世界观/03_power_system/` 力量体系是否矛盾
- 与已有各模块设定是否冲突
- 术语命名是否符合规则（『』【】标记）

### 6. 输出
设定写入 `01_世界观/` 对应子目录。
更新 `01_世界观/INDEX.md`。

## 注意
- 世界观信息通过正文间接传递，不能直接"灌设定"
- 新设定必须包含"在故事中如何自然揭示"的建议
- 命名遵循 A+B+C 三层结构，适当使用生僻字
