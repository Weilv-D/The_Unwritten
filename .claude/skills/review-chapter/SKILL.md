# 审查章节

对已完成章节进行七维度质量评审。

## 必读规则
- `core-style.md`（FAIL级硬规则、重点改进项、视角纪律）
- `core-character.md`（角色自主性、情感克制）

## 输入参数
章节编号：$ARGUMENTS（格式：卷1/ch003）

## 执行步骤

### 1. 加载素材
- `05_正文/$ARGUMENTS.md` 正文内容
- `04_章纲/$ARGUMENTS.md` 对应章纲

### 2. 七维度评估

| 维度 | 检查要点 |
|------|---------|
| 剧情 | 场景推演公式是否完整？转折是否自然？钩子是否有力？ |
| 角色 | 行为是否符合五维设定？对话有潜台词？语音可区分？ |
| 场景 | 环境描写在为叙事服务？氛围与基调匹配？ |
| 对话 | 每句对话都有功能？无注水闲聊？信息差被利用？ |
| 节奏 | 张弛有度？三幕比例合适？信息密度均匀？ |
| 文笔 | 半文半白？无现代词汇？无辞藻堆砌？实词有典可查？虚词用法合语法？段落可复述核心信息？ |
| 逻辑 | 时间线合理？视角纪律？无设定矛盾？ |

### 3. FAIL级硬规则复查
逐条检查 `core-style.md` 中定义的 FAIL 级硬规则：
- 每条标注 PASS/FAIL
- FAIL 项给出具体位置和修改建议

### 3.5 深审项（按需）

若作者要求终稿级深审，再执行 `core-style.md` 的深审项：
- 标注所有不确定出处的实词为“待查证”
- 检查虚词是否有语法依据
- 标注所有新造复合名词
- 用白话复述难段，标记“可读性不足”

### 4. 输出报告

将报告写入：`06_追踪/审查报告/卷X_chNNN_章节审查.md`

```
## 章节审查报告
**章节**：$ARGUMENTS

### 七维度评分
| 维度 | 评分(1-10) | 说明 |
|------|-----------|------|
| 剧情 | X | ... |
| 角色 | X | ... |
| [其他维度] | X | ... |

### FAIL级硬规则
| # | 规则 | 状态 |
|---|------|------|
| 1 | 术语与设定一致性 | PASS/FAIL |
| [其余硬规则] | | |

### 亮点
- [列出做得好的地方]

### 必须修复
- [列出 FAIL 项和关键问题]

### 建议优化
- [列出可选的改进建议]

**综合评分**：X/10
```

### 5. 询问
询问作者是否需要根据报告修改。如需要，可运行 `/edit-chapter`。
