# 审查章节

## 必读

- `.claude/rules/core-style.md`
- `.claude/rules/core-character.md`
- `06_追踪/审查报告/审查报告模板.md`

## 输入

章节编号：`$ARGUMENTS`，格式如 `卷1/ch003`。

## 流程

1. **加载上下文清单**
   - `05_正文/$ARGUMENTS.md`
   - `04_章纲/$ARGUMENTS.md`
   - 章纲“来源清单”对应的设定、人物、伏笔文件。
   - 缺正文或章纲时停止，不做空泛审查。

2. **证据优先审查**
   - 先列“必须修复”，再列“建议优化”，评分只放报告末尾。
   - 每个问题必须包含：问题位置、触发规则、证据摘录或概述、修改建议、需作者决策项。
   - 不确定项标注“待查证”，不得伪装为确定结论。

3. **FAIL级硬规则**
   - 术语与设定冲突。
   - POV 越界。
   - 说明书式设定倾倒。
   - 角色或势力为推剧情降智。
   - 明显现代口语、网络语、现代社科套话。
   - 关键冲突被私自消解。

4. **七维度复查**
   - 剧情：场景公式是否完整，转折是否自然。
   - 角色：行为是否符合五维设定，对话是否有潜台词。
   - 场景：环境是否服务叙事。
   - 对话：是否利用信息差，是否注水。
   - 节奏：张弛、信息密度、章节收束。
   - 文笔：半文半白、古典质感、段落可复述。
   - 逻辑：时间线、能力边界、伏笔操作。

5. **深审项**
   - 仅在作者要求终稿或精修时执行词源、虚词、名词与可读性深审。

6. **输出**
   - 按 `审查报告模板.md` 写入 `06_追踪/审查报告/卷X_chNNN_章节审查.md`。
   - 报告末尾列“建议下一步”：直接修订、补设定、请作者决策或可暂缓。
