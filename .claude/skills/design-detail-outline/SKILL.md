# 设计卷次细纲

从大纲展开为逐章细纲，具体到每章事件。

## 必读规则
- `core-plot.md`（微观场景引擎、六大核心场景类型）

## 输入参数
卷次：$ARGUMENTS（如 卷1）

## 执行步骤

### 1. 加载素材
- `03_大纲/卷次/$ARGUMENTS_大纲.md`（卷大纲）
- `03_大纲/故事梗概.md`
- `01_世界观/` 相关设定
- `02_人物/人物速查.md`
- 已有的上卷细纲（如有）

如果大纲不存在，建议先运行 `/design-outline $ARGUMENTS`。

### 2. 确认设计方向

**与作者讨论确认**：
- 本卷总章数
  - 若规模尚未锁定，先提供章节规模建议（如紧凑卷 24-36 章、标准卷 40-60 章、长卷 60 章以上）
- 支线穿插节奏
- 有无必须包含的关键场景
- 节奏偏好（前松后紧/均匀/高潮集中）

### 3. 先锁定章节规模

如果本卷规模未锁定，先给出 2-3 个可行方案，每个方案包含：
- 章节总量区间
- 适合的叙事密度
- 对支线容量与高潮展开的影响
- 推荐方案及理由

在作者锁定规模前，只输出**章节范围建议和分布草案**，不要把 `001-100` 当作固定模板。

### 4. 十大序列分配章节范围

在规模锁定后，再把十大序列映射到具体章节区间。默认用“占比 + 对应章数范围”表达，而不是先验固定章数。

示例格式：

```
序列        占比参考     章节范围     说明
开场        5%-10%      ch001-ch00X  建立局势与视角
切入        10%-15%     ch00X-ch00Y  冲突正式接入
……
高潮        10%-15%     ch00X-ch00Y  核心爆发
结局        5%-10%      ch00Y-ch00Z  余波与转接
```

### 5. 逐章细纲

为每章生成（300-500字/章）：
- **核心事件**：本章发生了什么
- **视角人物**：谁在叙述
- **信息释放**：表层/中层/深层各一层
- **悬念钩子**：章头小悬念 + 章尾大悬念
- **支线进度**：本卷支线的推进位置
- **正负主题**：上升或下降

### 6. 支线编排

每条支线标注：
- 出场章节
- 信息揭示节点
- 与主干的交汇点
- 收束章节

### 7. 输出
- 细纲写入 `03_大纲/卷次/$ARGUMENTS_细纲.md`
- 更新 `03_大纲/INDEX.md`

## 错误处理
- **大纲缺失或过于粗略**：停止并建议先完善大纲
- **章节间逻辑断裂**：标记断点，请作者确认过渡设计
- **与已有章纲冲突**：标记冲突，不自行覆盖
