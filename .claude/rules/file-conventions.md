# 文件命名与项目规范

## 目录结构

```
TheUnwritten/
├── 00_项目基础/    # 项目管理（任务、进度、决策）
├── 01_世界观/      # 世界观设定（按类别组织）
├── 02_人物/        # 角色档案（前缀分类）
├── 03_大纲/        # 大纲体系（梗概+卷次/大纲+细纲）
├── 04_章纲/        # 章节大纲（卷1/ch001.md 格式）
├── 05_正文/        # 正文章节（卷1/ch001.md 格式）
├── 06_追踪/        # 创作日志+伏笔追踪+审查报告
├── 07_*/           # 顶层编号预留位，当前未启用
└── 08_灵感/        # 灵感碎片+研究资料
```

## 文件命名规范

- 世界观：按 `01_世界观/NN_中文目录/NN_主题.md` 组织，目录编号必须唯一
- 人物：`前缀_姓名.md`（如 `主角_姓名.md`、`反派_某某.md`）
- 大纲：`03_大纲/卷次/卷X_大纲.md`（粗纲）、`03_大纲/卷次/卷X_细纲.md`（细纲）
- 章纲：`卷X/chNNN.md`（如 `卷1/ch003.md`）
- 正文：`卷X/chNNN.md`（与章纲对应）

## INDEX.md 维护规则

- 一级内容目录 `01_世界观`、`02_人物`、`03_大纲`、`04_章纲`、`05_正文`、`06_追踪`、`08_灵感` 均必须维护 `INDEX.md`
- 顶层 `07_*` 当前为预留编号位；如后续启用，应先在本文件和 `README.md` 中补说明，再落目录
- 每次创建/删除文件时，必须更新对应目录的 INDEX.md
- INDEX.md 中标注每个文件的创建状态
- 保持 INDEX.md 的查找表格式

## Skills 规范

- 项目自定义命令统一放在 `.claude/skills/<skill-name>/SKILL.md`
- workflow skill 默认通过 `/skill-name` 手动触发
- 参考型 skill 可按需要允许 Claude 自动加载

## 设定文件写法规范

- **不要"修改记录"段落**：Git负责版本追踪，文件末尾不写修改记录
- **不要补丁式口吻**：设定只写"是什么"，不写"不是什么"、不写"而非XX"、不标注"此处新增""此处修改"。新增内容直接融入上下文，不做对比、不做转折、不做说明性注释。
- **不要稿次标注**：不写"二稿""重写"，文件就是当前版本
- **不要总结变更**：改完不列"核心变化"清单，用户看得到文件

## Git 提交规范

```
feat(vol1/ch001): 完成第001章初稿（7500字）
fix(世界观): 修正力量体系等级描述不一致
docs(人物): 新增主角_姓名角色档案
review(vol1): 完成第一卷一致性审查
```

## 字数标准

- 正文每章：7500字（最低6500，最高8500）
- 章纲每章：300-800字
- 细纲每章：300-500字
- 粗纲每卷：2000-5000字
