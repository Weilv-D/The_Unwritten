# 人物设计规则

## 核心原则

### 角色五维定义
每个重要角色必须定义：
1. **表层身份**：社会角色（弟子/长老/城主/散修）
2. **隐藏身份**：真实目的或秘密（如有）
3. **核心欲望**：驱动一切的底层动机
4. **致命弱点**：性格缺陷或软肋
5. **关键关系节点**：3-5个（血缘/同盟/对抗/隐患）

### 动机驱动
- 人物的重大行动必须源自核心欲望
- 冲突来源于欲望碰撞，而非情节需要
- 无纯粹善恶，所有行为有利益或情感根基

### 配角分类部署
- **推动型**：在关键时刻推动主线发展
- **持续型**：贯穿故事，提供稳定性
- **伏笔型**：早期埋设，后期回收

### 人物弧光
- 每个角色有完整的成长/堕落轨迹
- 弧光类型：正面（克服缺陷）/ 负面（被缺陷吞噬）/ 平坦（坚持信念改变世界）
- 关键转变必须有触发事件

## 角色档案结构
使用 `02_人物/角色模板.md` 创建档案，包含：
- 基本信息、性格分析、内在需求
- 背景故事、对话风格、人物弧光
- 核心关系图谱

## 对话风格差异化
- 每个人物的说话方式必须可区分
- 读者仅通过对话就能识别是谁在说话
- 对话风格必须与性格和背景一致
- 仙侠世界：称谓严格体现等级
