# 日常写作流程

> 每日创作工作流程，基于写作要求.md的风格标准

---

## 每日启动流程（15分钟）

### 1. 读取进度
```
读取 00_项目基础/进度追踪.md
```
了解当前进度和目标。

### 2. 读取当前任务
```
读取 00_项目基础/任务清单.md
```
确定今天的具体写作目标。

### 3. 加载上下文
**必须读取**：
- 当前章节的章纲（`04_章纲/卷X/chYYY.md`）
- 当前出场人物的速查（`02_人物/人物速查.md`）
- 当前场景涉及的世界观设定（`01_世界观/` 相关文件）

**按需读取**：
- 前一章的正文（`05_正文/卷X/chYYY.md`，用于衔接）
- 相关伏笔记录（`06_追踪/伏笔追踪.md`）

> 提示：也可以直接使用 `/daily-start` 斜杠命令自动执行以上流程。

---

## 创作流程（2-4小时）

### 步骤1：确认章节目标
从章纲中确认本章的：
- 核心叙事功能（推动情节/揭示信息/展现代价/深化主题）
- 三幕结构（钩子→博弈→收束）
- 出场人物和场景设定

### 步骤2：检查风格标准
写作前快速回顾 `CLAUDE.md` 和 `.claude/rules/writing-style.md` 中的：
- 12条禁止规则
- 段落公式：[景-人-心]三段式
- 内心独白规范（『』包裹）

### 步骤3：写作正文
严格遵循写作要求.md中的风格规范：

**开场**（20%篇幅）：
- 3句话内建立场景、绑定视角、抛出本章核心悬念
- 先用环境渲染（阴沉/肃杀/宏大/幽静/诡谲）

**发展**（60%篇幅）：
- 通过对话、行动、视角切换推进核心事件
- 高频使用内心独白（『』包裹）
- 遵循[景-人-心]段落公式
- 实现人物状态实质性变化

**收束**（20%篇幅）：
- 给出阶段性结果
- 埋下新伏笔
- 结尾句需有余韵

### 步骤4：自检清单
完成章节后，逐项检查：

- [ ] 无直接情绪告知（"他很愤怒"类表述）
- [ ] 无作者评论或旁白说明
- [ ] 场景内视角固定（无上帝视角切换）
- [ ] 对话有信息推进（无闲聊）
- [ ] 内心独白使用『』包裹
- [ ] 遵循[景-人-心]段落结构
- [ ] 专用术语使用『』或【】包裹
- [ ] 半文半白语言风格
- [ ] 字数达标（约6000字）

> 提示：也可以使用 `/write-chapter 卷X/chYYY` 自动执行创作流程。

---

## 结束流程（30分钟）

### 1. 更新章节元数据
在正文文件头记录：章节号、字数、完成日期、视角人物

### 2. 更新追踪表
- 更新人物出场追踪（`02_人物/人物速查.md`）
- 更新伏笔追踪表（`06_追踪/伏笔追踪.md`）
- 更新创作日志（`06_追踪/创作日志.md`）

### 3. 更新进度
```
更新 00_项目基础/进度追踪.md
```

### 4. 记录灵感
如有新灵感或需要调整的内容，记录到：
```
08_灵感/INDEX.md 或相关文件
```

### 5. Git提交
```bash
git add 05_正文/卷X/chYYY.md 06_追踪/
git commit -m "feat(volX/chYYY): 完成第YYY章初稿（XXXX字）"
```

## 质量标准速查

### 必须做到
- 展示而非告知（Show, Don't Tell）
- 内心独白驱动叙事
- [景-人-心]段落结构
- 信息密度最大化
- 因果链严密

### 绝对禁止
- 直接告知情绪
- 作者评论/说教
- 视角混乱
- 术语矛盾
- 对话闲聊
- 儿戏战斗
- 人物降智
- 冗长拖沓
