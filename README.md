# The Unwritten

> Claude Code 协作长篇仙侠小说创作工作台

---

## 项目定位

本仓库用于组织长篇仙侠小说的设定、人物、大纲、章纲、正文与追踪资料，并为 Claude Code 提供稳定的项目上下文。

- **仓库用途**：创作工作空间，不是具体作品成稿
- **当前状态**：工作台已整治，具体作品规格仍待锁定
- **现有资料**：世界观基础资料已建档，人物/大纲/正文尚未启动
- **创作边界**：本次维护只优化工作流与信息架构，不替代作者锁定作品设定

---

## Claude Code 接口

| 接口 | 作用 |
|------|------|
| `CLAUDE.md` | 项目级运行时记忆入口，提供最小必要协作约束 |
| `.claude/rules/**/*.md` | 常驻或按路径加载的规则文件 |
| `.claude/skills/*/SKILL.md` | 项目自定义 skills；本仓库唯一的自定义命令入口 |
| `.claude/settings.local.json` | 当前仓库跟踪的项目配置特例，承载 hooks 与权限配置 |

> 说明：Claude Code 官方通常将团队共享配置放在项目共享 settings 文件中。本仓库出于历史兼容，继续版本控制 `.claude/settings.local.json`；这是项目特例，不代表官方默认推荐作用域。

## Skills 与自动化

- 项目自定义 workflow 统一放在 `.claude/skills/`，不再新增 `.claude/commands/` 兼容层。
- 除 `wenyan-writing` 外，项目 workflow skills 默认通过 `/skill-name` 手动调用。
- 当前 hooks 只有三项：敏感文件保护、创作日志记录、正文字数检查。
- 仓库**不会**在每次编辑后自动执行 Git 提交。
- 修改 `CLAUDE.md` 后，当前已打开的会话不会回溯替换旧缓存；要让新版项目记忆完整生效，请开启新会话。

## 审计入口

维护结构与引用一致性时，使用以下命令：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\audit-structure.ps1
```

该脚本会检查：

- 缺失的一级目录 `INDEX.md`
- 非规范 skill 入口文件
- 旧目录名与失效路径残留
- 不存在的规则/资料引用
- 过时的数量承诺与错误术语
