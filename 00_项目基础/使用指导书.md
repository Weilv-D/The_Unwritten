# TheUnwritten 项目使用指导书

> 本文档面向人类维护者，说明当前仓库的真实结构、Claude Code 对齐方式、skill 使用方式和日常维护入口。

---

## 一、项目概览

TheUnwritten 是一个**用 Claude Code 协作长篇仙侠小说创作的工作台**。

- **仓库角色**：承载设定、人物、大纲、章纲、正文与追踪资料
- **当前阶段**：工作台已完成结构整治，具体作品规格尚未锁定
- **协作方式**：人类负责创作决策，Claude 负责流程化辅助、整理、校验与执行
- **文档驱动**：所有资料以 Markdown 管理，Git 只负责版本追踪，不承担工作流解释

---

## 二、与 Claude Code 官方模型的对齐方式

- 项目运行时记忆入口使用 `CLAUDE.md`
- 项目规则使用 `.claude/rules/**/*.md`
- 项目自定义命令与工作流统一使用 `.claude/skills/*/SKILL.md`
- 当前仓库**不再新增** `.claude/commands/` 兼容层

> 项目特例：Claude Code 官方通常将团队共享配置放在项目共享 settings 文件中。本仓库因历史兼容，继续版本控制 `.claude/settings.local.json`。请将它视为“项目配置特例”，而不是通用推荐做法。

## 四、文档职责

| 文档 | 职责 |
|------|------|
| `README.md` | 仓库入口与当前状态说明 |
| `CLAUDE.md` | 项目级运行时记忆，只保留最小协作约束 |
| `00_项目基础/统一方法论与工作流指导.md` | 方法论与 S.O.P 权威源 |
| `00_项目基础/使用指导书.md` | 人类维护手册 |

原则：

- `README.md` 负责“这个仓库是什么”
- `CLAUDE.md` 负责“Claude 进入仓库时需要知道什么”
- 方法论文档负责“创作流程与质量标准是什么”
- 本指导书负责“日常如何维护与使用”

## 五、规则体系

当前项目规则只有以下 5 个：

| 规则文件 | 主要用途 |
|----------|----------|
| `core-style.md` | 文风、视角纪律、14 条禁忌红线 |
| `core-character.md` | 角色方法、欲望与弧光约束 |
| `core-plot.md` | 三幕结构、节拍、场景引擎 |
| `core-world.md` | 世界观展开、术语、势力动力学 |
| `file-conventions.md` | 命名、索引、目录规范 |

说明：

- 规则通过 `.claude/rules/` 加载
- 其中可按 `paths` 限定适用目录
- 规则描述的是“常驻知识”；具体工作流请走 skills

## 七、Hooks 与配置

当前 hooks 只有三项，定义在 `.claude/settings.local.json`：

| Hook | 时机 | 真实行为 |
|------|------|----------|
| `guard-sensitive.sh` | `PreToolUse` | 阻止误改敏感文件 |
| `log-change.sh` | `PostToolUse` | 记录创作相关文件变更 |
| `word-count-check.sh` | `PostToolUse` | 对 `05_正文/` 做字数检查 |

注意：

- hooks 会自动执行
- hooks 不会替你自动提交 Git
- 修改 hooks 后，应通过 `/hooks` 或重启会话确认加载的是最新配置快照

## 八、日常使用流程

### 开始一次创作会话

1. 运行 `/daily-start`
2. 先查看目标目录的 `INDEX.md`
3. 再进入具体设定、人物、大纲或章纲文件

### 规划与创作

1. 作品尚未锁定时，先补 `03_大纲/故事梗概.md`
2. 世界观补充走 `/design-world`
3. 人物建档走 `/create-character`
4. 卷规划走 `/design-outline` / `/design-detail-outline`
5. 章节规划走 `/design-chapter-outline`
6. 正文创作走 `/write-chapter`

### 审查与收束

1. 单章审查走 `/review-chapter`
2. 整卷审查走 `/review-volume`
3. 矛盾排查走 `/check-consistency`
4. 图谱同步走 `/kg-update`
5. 会话结束走 `/end-session`

## 九、审计与维护

结构变更、目录重命名或批量修文后，运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\audit-structure.ps1
```

审计项包括：

- 一级目录是否缺失 `INDEX.md`
- skills 是否全部使用规范入口 `SKILL.md`
- 旧目录名与失效路径是否残留
- 是否仍引用不存在的规则或资料文件
- 是否仍出现过时数量承诺与旧术语
