# TheUnwritten 项目使用指导书

> 本文档是项目的完整使用指南。覆盖目录结构、规则体系、技能系统、自动化钩子、知识图谱，以及按工作流阶段的操作方法。

---

## 一、项目概览

TheUnwritten 是一个用 Claude Code 辅助创作的仙侠小说项目。核心思路：

- **人机协作**：人类拍板创作决策，AI 提供方案、填充细节、检查一致性
- **文档驱动**：所有设定、大纲、正文均以 Markdown 文件存储，Git 版本管理
- **技能化工作流**：通过 `/技能名` 触发标准化创作流程
- **规则约束**：14条禁止规则 + 风格规范 + 方法论，确保输出质量

---

## 二、目录结构

```
TheUnwritten/
├── 00_项目基础/        # 项目管理（进度、任务、本指导书）
├── 01_世界观/          # 世界观设定（按类别子目录组织，有 INDEX.md）
├── 02_人物/            # 角色档案（前缀_姓名.md，有 INDEX.md）
├── 03_大纲/            # 大纲体系（梗概 + 卷次大纲/细纲，有 INDEX.md）
├── 04_章纲/            # 章节大纲（卷X/chNNN.md）
├── 05_正文/            # 正文章节（卷X/chNNN.md，与章纲对应）
├── 06_追踪/            # 创作日志 + 伏笔追踪 + 知识图谱
├── 08_灵感/            # 灵感碎片 + 研究资料（有 INDEX.md）
└── .claude/
    ├── CLAUDE.md       # AI 主指令文件（最高优先级）
    ├── settings.json   # 权限 + hooks + MCP 配置
    ├── rules/          # 10个规则文件（按路径自动激活）
    └── skills/         # 16个技能（按触发词或 /命令 激活）
```

## 三、规则体系

规则文件位于 `.claude/rules/`，通过 `paths` 字段自动激活——编辑对应目录的文件时，相关规则自动加载到上下文。

| 规则文件 | 激活路径 | 作用 |
|----------|----------|------|
| `writing-style.md` | 正文、章纲 | 半文半白语言、色彩材质、术语标记、段落公式、战斗公式 |
| `forbidden-rules.md` | 正文、章纲 | 14条禁止规则（含判定标准），写作后逐条自检 |
| `creative-philosophy.md` | 始终加载 | 10条创作哲学（客观视角、展示非告知、信息差等） |
| `methodology-world.md` | 世界观、人物 | 势力分层博弈法、道统人格化、历史包袱驱动 |
| `methodology-plot.md` | 大纲、章纲 | 对话即剧情、小剧情公式、场景设计五步法、反派叙事定位 |
| `methodology-writing.md` | 正文 | 配角五原则、战斗三轴驱动、服务式视角、情感隐忍克制、信息控制六原则 |
| `file-conventions.md` | 所有目录 | 文件命名规范、INDEX.md 维护、Git 提交格式、字数标准 |
| `character-rules.md` | 人物 | 人物创建流程与模板规范 |
| `outline-rules.md` | 大纲 | 大纲设计流程规范 |
| `worldbuilding-rules.md` | 世界观 | 世界观构建流程规范 |

**读取优先级**：CLAUDE.md > rules/（按 paths 自动匹配） > skills/（按触发激活）

## 四、技能系统（/命令）

技能位于 `.claude/skills/`，通过说触发词或输入 `/技能名` 激活。

### 项目管理

| 命令 | 触发词 | 用途 |
|------|--------|------|
| `/daily-start` | 开始、开工、进度、到哪里了 | 加载项目状态、统计字数、确认今日目标 |
| `/end-session` | 收工、结束、保存进度 | 保存进度、记录待办、建议下次目标 |
| `/init-project` | 初始化、重置项目 | 检查并创建所有缺失的基础设施文件 |

### 设定构建

| 命令 | 触发词 | 用法 |
|------|--------|------|
| `/design-world` | 世界观、设定、地理、势力 | `/design-world 势力` 或 `/design-world 地理` |
| `/create-character` | 创建人物、新角色、人物设计 | `/create-character 主角_姓名` |

### 大纲规划

| 命令 | 触发词 | 用法 |
|------|--------|------|
| `/design-outline` | 大纲、粗纲 | `/design-outline 卷1` |
| `/design-detail-outline` | 细纲、详细大纲 | `/design-detail-outline 卷1` |
| `/design-chapter-outline` | 章纲、章节大纲 | `/design-chapter-outline 卷1/ch003` |

### 正文创作

| 命令 | 触发词 | 用法 |
|------|--------|------|
| `/write-chapter` | 写正文、写一章、写第X章 | `/write-chapter 卷1/ch003` |

### 审查修改

| 命令 | 触发词 | 用法 |
|------|--------|------|
| `/review-chapter` | 审阅、审查、review | `/review-chapter 卷1/ch003` |
| `/edit-chapter` | 修改章节、编辑正文 | `/edit-chapter 卷1/ch003` |
| `/review-volume` | 卷次审查、全卷审查 | `/review-volume 卷1` |
| `/check-consistency` | 一致性、矛盾检查 | `/check-consistency 卷1/ch003` |

### 辅助工具

| 命令 | 触发词 | 用法 |
|------|--------|------|
| `/capture-idea` | 灵感、想法、闪念 | `/capture-idea 主角身世可以和某势力有关` |
| `/unstuck` | 卡文、写不下去、卡住了 | `/unstuck` |
| `/kg-update` | 知识图谱、更新图谱 | `/kg-update full` 或 `/kg-update 人物` |

## 五、自动化钩子（Hooks）

项目配置了三个钩子，编辑文件时自动运行：

| 钩子 | 触发时机 | 功能 |
|------|----------|------|
| `guard-sensitive.sh` | Write/Edit 执行前 | 检查是否包含敏感信息，阻止误写 |
| `log-change.sh` | Write/Edit 执行后 | 自动记录文件变更日志 |
| `word-count-check.sh` | Write/Edit 执行后 | 自动统计字数，检查是否达标 |

这些钩子无需手动操作，编辑正文或设定文件时会自动触发。

## 六、知识图谱（Neo4j）

项目集成了 Neo4j 知识图谱，用于追踪人物关系、世界观设定和伏笔。

- **同步方式**：`/kg-update` 技能扫描项目文件批量同步
- **查询方式**：直接对话提问（如"查一下某某和某某的关系"）
- **同步范围**：`full`（全量）/ `人物` / `世界观` / `伏笔` / `场景`

详细 Schema 和使用指南见 `06_追踪/知识图谱.md`。

## 七、标准工作流

### 从零开始的新项目

```
/init-project          → 初始化项目结构
/design-world [目标]    → 构建世界观（涟漪展开法）
/create-character [名]  → 创建主要人物
/design-outline 卷1     → 设计卷大纲（节拍器理论）
/design-detail-outline 卷1 → 展开细纲
/design-chapter-outline 卷1/ch001 → 生成章纲
/write-chapter 卷1/ch001 → 创作正文（7500字）
```

### 每日创作流程

```
/daily-start           → 加载进度、确认目标
/write-chapter ...     → 创作
/review-chapter ...    → 审阅
/edit-chapter ...      → 修改
/end-session           → 保存进度
```

### 设定迭代流程

```
/design-world [新设定]  → 扩展世界观
/kg-update full         → 同步到知识图谱
/check-consistency 卷1  → 检查一致性
```

## 八、协作原则

| 场景 | AI 行为 | 你的角色 |
|------|---------|----------|
| 创作决策 | 提供 2-3 个备选方案 | 拍板选定 |
| 技术填充 | 基于章纲自由发挥场景/对话/节奏 | 审阅并反馈 |
| 一致性检查 | 自动扫描跨文件矛盾 | 确认修复方案 |
| 遇到不确定 | 提供判断 + 建议方案，最多追问一次 | 一次性反馈 |
| 初稿质量 | 初稿即达文学标准，不写凑字数草稿 | 审阅并指出方向问题 |

## 十一、常见问题

**Q：新会话开始时怎么快速恢复上下文？**
用 `/daily-start` 或直接说"开始"，AI 会自动读取项目状态文件并汇报进度。

**Q：AI 一直追问不停怎么办？**
直接告诉它"按你判断推进，不要追问"。CLAUDE.md 的工作推进原则要求 AI 优先推进、减少追问。

**Q：术语写错了怎么办？**
用 `/check-consistency` 检查，或直接指出错误术语，AI 会搜索全项目并同步修正。

**Q：灵感突然来了怎么办？**
说"灵感"或 `/capture-idea [内容]`，自动记录并归档到 08_灵感 目录。

**Q：写不下去怎么办？**
说"卡文"或 `/unstuck`，AI 分析当前进度并提供多个推进方向。

**Q：想修改已经写好的章节？**
用 `/edit-chapter 卷1/ch003`，然后说明要改什么。

**Q：MCP 工具报错了？**
直接告知 AI 切换方案。连续失败 2 次后 AI 会自动停止重试。
