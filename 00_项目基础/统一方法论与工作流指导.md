# 统一方法论与工作流规范

> 本文件是项目唯一的**方法论权威源**，定义创作阶段的理论框架、执行流程与质量标准。
> 仓库当前是创作工作台，具体作品规格仍待锁定；本文件描述的方法独立于单一作品而存在。
> 具体的风格、角色、世界观、剧情法则分别由 `core-style.md`、`core-character.md`、`core-world.md`、`core-plot.md` 承载，本文件负责说明这些规则在仓库中的层级与使用方式。

---

## 一、规则分层

为避免把经验模板误当成绝对铁律，项目中的要求统一分为三层：

| 层级 | 含义 | 典型内容 |
|------|------|---------|
| 硬规则 | 违反即直接损害成稿质量或一致性，必须优先满足 | POV 限知、术语复用、设定一致性、因果连贯、角色动机自洽、禁止明显现代口语、禁止说明书式设定倾倒、发现关键冲突必须停下 |
| 默认模板 | 常用且高效的工作模板，优先采用，但可根据卷规模、章节功能、叙事密度偏离 | 三幕比例、章节规模建议、每章默认字数、章纲/细纲长度、常用检查顺序 |
| 参考技法 | 用于启发和拔高，不作为失败条件 | 词汇清单、场景类型示例、讨喜范式、展示优先的示例写法、文言细查清单 |

使用原则：

1. 先保硬规则，再谈模板优化，再做风格拔高。
2. 默认模板偏离时，要能说明原因；不因凑模板损伤成稿。
3. 参考技法用于提高上限，不用于机械判废。

## 二、核心统一定理

### 1. 剧情与角色的逻辑融合

- **定理**：剧情的大势倒推并非由作者强加，而是由**世界观制度的残酷性**与**高层势力的博弈**所决定的客观局势。
- **展现**：角色的行动不受剧情强制扭曲，而是基于自身欲望、弱点、关系和认知盲区做选择。高潮或悲剧的成立，来自角色忠于自我后与大势相撞。

### 2. 大纲架构的分层法则

项目仍采用“宏观→中观→微观”的分层框架，但其中的比例和节拍默认视为**模板**，不是不可偏离的律法。

| 层级 | 名称 | 工具 | 适用范围 | 说明 |
|------|------|------|---------|------|
| 战略层 | 逆向锚定 | 逆向五步法 | 全书/单卷规划 | 只定起止状态与关键不可动摇节点 |
| 战役层 | 比例切分 | 三幕式 | 单卷结构 | 默认按 20/60/20 估算节奏，可按卷规模偏离 |
| 战术层 | 节拍编排 | 十大序列 | 单卷细纲 | 用于分配叙事功能和波峰，不强行绑定固定章数 |
| 微观层 | 场景推演 | 场景公式 | 单章章纲 | 用于保障单章因果推进与信息差 |

## 三、逆向拆解法（战略层工具）

设计剧情弧线时，先从终点倒推，确保因果链严密。

1. **确定终点**：这一段故事结束后，世界或主角发生了什么质变？
2. **反推关键节点**：达成终点必须发生哪些事件？列出 3-5 个核心节点。
3. **分配角色任务**：每个节点由谁完成？为何必须是 ta？
4. **设置障碍与转机**：每个节点的阻碍是什么？转机是否体现角色特质？
5. **正向验证**：从起点顺推，是否仍保有悬念和意外感？

## 四、角色创作流程（六步法）

从结构到细节系统化构建角色。底层逻辑详见 `core-character.md`。

1. **位置与生态位**：先确定角色在权力、资源、血缘结构中的位置。
2. **欲望与恐惧**：明确生存/位置/超越三层欲望及其冲突。
3. **范式选择**：选择最适合该角色功能的表达范式。
4. **弧光与代价**：确定成长路径与必须支付的代价。
5. **关系编织**：把角色放入已有关系网，而不是孤立造人。
6. **细节赋予**：补足语言、动作、物件等可复用识别点。

## 五、创作管线闭环 S.O.P

为解决进度孤岛、伏笔遗忘和流程漂移，所有创作动作都应执行**后置同步**，但同步动作必须建立在真实状态而非机械模板上。

### 阶段 0：启动同步（Daily Start） `/daily-start`

- **动作**：回顾 `00_项目基础/进度追踪.md`、`00_项目基础/任务清单.md`、近期 `06_追踪/创作日志.md`，统计 `05_正文/` 实际字数。
- **要求**：目标字数未锁定时显示“未锁定”，不硬算百分比；当前阶段由实际文件存在情况判断。

### 阶段 1：要素建档（World / Character Sync）

- **动作**：任何新势力、新地域、新出场重要人物，优先在 `01_世界观/` 或 `02_人物/` 建档。
- **后置**：同步更新相关 `INDEX.md`。

### 阶段 2：卷纲统筹（Volume Outline） `/design-outline` → `/design-detail-outline`

- **动作**：依据分层法则输出整卷粗纲与细纲。
- **要求**：卷规模未锁定时，先给章节范围建议，不把 `100` 章当作默认前提。
- **后置**：将主要伏笔摘录到 `06_追踪/伏笔追踪.md`，但仅登记已明确锁定的伏笔链。

### 阶段 3：章纲推演（Chapter Outline） `/design-chapter-outline`

- **动作**：用场景公式和信息差原则推演单章。
- **要求**：每章至少说清楚“目标、阻碍、意外变量、决断转折”；输出“写作契约”，列明 POV、信息释放、禁写事项与需查证来源；三幕与字数分配可按章功能偏离默认值。

### 阶段 4：正文渲染（Drafting） `/write-chapter`

- **动作**：按“上下文取证 → 连续性/设定预检 → 草稿 → 反思修订 → FAIL 红线自检 → 作者确认后存档”的顺序产出正文，默认目标为 7500 字。
- **要求**：
  - 6500-8500 是常用参考区间，不是硬停机线。
  - 首稿优先检查连续性和 FAIL 级硬规则，不做重型词源审校。
  - 若字数偏离默认范围，说明原因，不靠注水或删肉来凑模板。
  - 短篇、练习、试笔必须先标注 canon 状态：`正式采纳 / 试笔参考 / 不入库`，不得自动写入世界观或人物事实。

### 写作质量闭环

写作输出采用两级质量闸门，避免把终稿审校压力前置到首稿阶段。

1. **首稿闸门**：只抓连续性、设定一致、POV 限知、说明书式设定倾倒、明显现代口语、角色动机自洽。任一 FAIL 项触发，先修复红线，再谈文采。
2. **反思修订**：草稿完成后先列出 3-5 个高风险点，再对正文做一次定向修订；修订只处理实际问题，不为“更华丽”而堆词。
3. **终稿深审**：仅在作者要求精修或终稿时执行词源、虚词、句法、用典和可读性深审。深审发现不确定项时标注“待查证”，不得伪装成确定知识。
4. **证据优先**：审查报告先列问题位置、触发规则、修改建议和需作者决策项；评分只作为末尾参考。
5. **来源约束**：涉及仓库术语时必须回到源文件查证，不得只凭 `INDEX.md` 的一句释义扩写。

### 阶段 5：审查与归档（Review） `/review-chapter` / `/review-volume` / `/check-consistency`

- **动作**：执行单章审查、卷次审查或一致性检查。
- **要求**：
  - 单章与卷次审查报告必须落盘到 `06_追踪/审查报告/`
  - 一致性检查先产出“发现的问题”和“建议更新的追踪项”，经作者确认后再回写追踪文件
  - 报告采用证据优先格式：必须修复项在前，建议优化在后，综合评分放末尾
  - 深审项用于终稿或精修阶段，不强压到首稿

### 阶段 6：会话收束（End Session） `/end-session`

- **动作**：以当前 Git 变更和最近日志共同回顾成果、待办和下次入口。
- **要求**：`创作日志.md` 是辅助日志，不作为唯一会话事实来源。

### 辅助技能（按需调用）

| 技能 | 用途 |
|------|------|
| `/capture-idea` | 快速记录灵感碎片，归档到 `08_灵感/` |
| `/check-consistency` | 跨文件一致性检查（角色/世界观/时间线/伏笔/细节） |
| `/review-chapter` | 单章质量评审并落盘报告 |
| `/review-volume` | 全卷综合审查并落盘报告 |
| `/edit-chapter` | 基于反馈修订已有章节 |
| `/unstuck` | 分析卡点，提供推进方向 |
| `/kg-update` | 扫描项目文件，同步实体到 Neo4j |
| `/init-project` | 扫描并补齐基础设施文件 |

## 六、创作检查清单

### 硬规则检查

- [ ] POV 是否限知？是否有越界信息？
- [ ] 术语是否与源文件一致？
- [ ] 设定、人物、时间线是否自洽？
- [ ] 角色选择是否能追溯到欲望、弱点和位置？
- [ ] 是否出现说明书式设定倾倒或明显现代口语？
- [ ] 若发现关键冲突，是否已停下而非私自消解？

### 默认模板检查

- [ ] 当前卷/章是否仍大体符合既定节奏？
- [ ] 偏离默认字数或模板时，是否有清晰理由？
- [ ] 伏笔、支线、高潮、收束是否有基本节拍安排？

### 参考技法检查

- [ ] 是否有足够的风格锚定与语言辨识度？
- [ ] 展示、留白、动作传情等手法是否用在该用的位置？
- [ ] 是否存在可在精修阶段继续拔高的段落？

## 七、常见误区

| 误区 | 表现 | 处理原则 |
|------|------|---------|
| 机械套模板 | 为了凑三幕比例、固定章数或固定字数而扭曲内容 | 模板可偏离，先保因果与质量 |
| 只写不记 | 正文推进了，但伏笔、进度、待办没有同步 | 每个阶段完成后最少做一次后置同步 |
| 只看日志 | 把 hook 日志当成全部会话事实 | 结合 Git 状态与日志共同回顾 |
| 过早重审 | 首稿阶段就做词源级、句法级深审 | 首稿先抓连续性与硬规则，深审后置 |
| 用技法判死刑 | 把词表、场景类型、展示示例当成硬规则 | 这些是启发与拔高，不是统一废稿线 |
