# 统一方法论与工作流规范

> 本文件是项目唯一的**方法论权威源**，定义所有创作阶段的理论框架、执行流程与质量标准。
> 具体的风格禁令、角色法则、世界观法则、剧情法则分别由 `core-style.md`、`core-character.md`、`core-world.md`、`core-plot.md` 承载，本文件不重复这些内容。

---

## 一、 核心统一定理

### 1. 剧情与角色的逻辑融合（消除宿命论与自由意志的悖论）
* **定理**：剧情的大势倒推（结局必然点）并非由作者强加，而是由**世界观制度的残酷性（吃人机制）**与**高层势力的博弈（棋手网罗）**所决定的客观局势。
* **展现**：角色的行动不受剧情强制扭曲，他们完全基于自身的**五维欲望（表层/隐藏身份、核心欲望、致命弱点、关系网）**行事。悲剧或高潮的发生，正是因为角色忠于自我（如贪婪者贪刀、重情者施救），不可避免地踏入局势的必然收束点。

### 2. 大纲架构的分层法则

废除多轨并行的相互矛盾的大纲理论，统合为以下立体结构。三者是**宏观→中观→微观**的分层关系，而非并行互斥：

| 层级 | 名称 | 工具 | 适用范围 | 说明 |
|------|------|------|---------|------|
| 战略层 | 逆向锚定 | 逆向五步法 | 全书/单卷规划 | 只定起止状态与2-3个不可动摇的历史级事件位点，不过度干预过程 |
| 战役层 | 比例切分 | 三幕式 (20/60/20) | 单卷结构 | 将卷切分为三幕，控制宏观节奏比例 |
| 战术层 | 节拍编排 | 十大序列 | 单卷细纲 | 将三幕细化为十大节拍（开场→切入→激动→进展→转折→二次转折→危机→导入→高潮→结局），分配章节范围 |
| 微观层 | 场景推演 | 场景公式 | 单章章纲 | `场景 = POV目标 + 阻碍/信息差 + 意外变量 + 决断转折` |

**三幕式与十大序列的关系**：
- 三幕式定义的是**比例**（第一幕20%、第二幕60%、第三幕20%）
- 十大序列定义的是**节拍**（每个序列占多少章、叙事密度如何）
- 映射关系：开场+切入+激动 = 第一幕 | 进展+转折+二次转折+危机+导入 = 第二幕 | 高潮+结局 = 第三幕

## 二、 逆向拆解法（战略层工具）

设计大剧情弧线的五步法——从终点倒推，确保因果链严密。

### 步骤

1. **确定终点**：这场大剧情结束后，世界/主角发生了什么质变？用一句话概括终点状态。
2. **反推关键节点**：要达成终点，必须发生哪些事件？按因果顺序列出3-5个关键节点。
3. **分配角色任务**：每个关键节点由谁完成？为什么必须是ta？ta有什么不可替代性？

| 关键节点 | 执行者 | 为何必须是ta |
|----------|--------|-------------|
|          |        |             |

4. **设置障碍与转机**：每个节点会遇到什么困难？如何解决？解决方式是否体现角色特质？

| 节点 | 障碍 | 转机 |
|------|------|------|
|      |      |      |

5. **正向验证**：从起点正向阅读，是否仍能保持悬念和意外感？如果提前能猜到结局，需要调整节点顺序或增加变量。

## 三、 角色创作流程（六步法）

从功能到细节，系统化构建角色。

1. **功能定位**：确定角色在故事中的核心功能（破局者/智谋者/守护者/锋刃/外交者/传承者/变数/旁观者）
2. **讨喜范式选择**：选择最适合该功能的讨喜范式（残缺守护者/负重凡人/清醒野心家/温柔强者/被误解者）
3. **欲望与矛盾设计**：确定核心欲望和内在矛盾，确保欲望驱动行动、矛盾制造张力
4. **弧光规划**：确定弧光类型（成长型/揭示型/复合型），设计3-4个转折点或揭示层
5. **关系编织**：将角色编织进已有关系网，确定利益/情感/权力/信息四个维度的关系
6. **细节赋予**：分配2-3个标志性细节（口头禅/习惯动作/物件等），确保贯穿始终

> 角色的五维核心定义（表层身份/隐藏身份/核心欲望/致命弱点/关键关系节点）详见 `core-character.md`。

## 五、 创作管线闭环 S.O.P（Standard Operating Procedure）

为解决进度孤岛与伏笔遗忘，所有创作动作必须执行**后置同步挂载**规则，严禁"只写不记"。

### 阶段 0：启动同步（Daily Start） `/daily-start`
* **动作**：回顾 `00_项目基础/进度追踪.md` 与 `00_项目基础/任务清单.md`，统计 `05_正文/` 字数。
* **目标**：不带偏见地建立今日上下文，确认上一环节遗留问题。

### 阶段 1：要素建档（World/Character Sync）
* **动作**：任何新势力、新地域、新出场重要人物，必须先在 `01_世界观` 或 `02_人物` 下建立档案。可使用 `/design-world` 或 `/create-character`。
* **后置**：更新 `02_人物/INDEX.md` 与 `01_世界观/INDEX.md`。

### 阶段 2：卷纲统筹（Volume Outline） `/design-outline` → `/design-detail-outline`
* **动作**：依据分层法则（逆向锚定→三幕式→十大序列），输出整卷粗纲与细纲（写入 `03_大纲`）。
* **后置（核心强制）**：必须将该卷埋设的三条以上主伏笔，摘录至 `06_追踪/伏笔追踪.md` 注册"未结伏笔"条目。

### 阶段 3：章纲推演（Chapter Outline） `/design-chapter-outline`
* **动作**：套用微观场景公式与多势力对峙原则，推演角色动机，确保无机械降神行为（写入 `04_章纲`）。
* **核对**：检查角色是否出现"为推剧情而降智/扭曲性格"的现象。

### 阶段 4：正文渲染（Drafting） `/write-chapter`
* **动作**：依据 `core-style.md` 中的文风要求及 **14条禁忌红线**，渲染7500字正文（写入 `05_正文`）。字数标准：最低6500，最高8500。

### 阶段 5：状态流转与归档（End Session） `/end-session`
* **动作**：写完正文后，强制执行状态同步：
  1. 更新 `00_项目基础/进度追踪.md` 的当前写足字数和章节。
  2. 划去 `00_项目基础/任务清单.md` 的当前完成项。
  3. 将正文中填坑的伏笔，在 `06_追踪/伏笔追踪.md` 中标记为 [已回收]。
  4. 将产生的新知识、关系网变化，推入 `06_追踪/知识图谱.md`（可通过 `/kg-update` 同步至 Neo4j）。

### 辅助技能（不绑定特定阶段，按需调用）

| 技能 | 用途 |
|------|------|
| `/capture-idea` | 快速记录灵感碎片，自动分类至 `08_灵感/` |
| `/check-consistency` | 跨文件一致性检查（角色/世界观/时间线/伏笔/细节） |
| `/review-chapter` | 单章七维度质量评审 |
| `/review-volume` | 全卷四阶段综合审查 |
| `/edit-chapter` | 基于反馈或审查修改已有章节 |
| `/unstuck` | 分析卡点，提供3-5个推进方向 |
| `/kg-update` | 扫描项目文件，同步实体至 Neo4j 知识图谱 |
| `/init-project` | 扫描并创建缺失的基础设施文件 |

## 六、 创作检查清单

### 剧情检查（5项）

- [ ] 三幕结构完整吗？每一幕有明确功能吗？
- [ ] 每个场景有不可替代性吗？（能否删掉/合并？）
- [ ] 伏笔有跨章节埋设吗？有追踪清单吗？
- [ ] 多线并行有主次之分吗？节奏有错峰吗？
- [ ] 张力曲线适合当前位置吗？（布局低→对抗升→高潮峰→收束降）

### 角色检查（5项）

- [ ] 每个重要角色的行动可追溯到核心欲望吗？
- [ ] 角色语音可区分吗？仅凭对话能识别是谁在说话？
- [ ] 弧光在按计划推进吗？转折/揭示节点到位？
- [ ] 共情触发已部署吗？读者有理由关心这个角色吗？
- [ ] 没有角色为了剧情方便而做出不符合性格的行为？

### 写作检查（5项）

- [ ] "展示而非告知"原则维持了吗？没有旁白解释角色情感？
- [ ] 对话有潜台词吗？说的和想的不一样？
- [ ] 视角纪律维持了吗？没有越界获取角色不该知道的信息？
- [ ] 环境描写在为叙事服务吗？（映射情绪/嵌入信息/营造张力/调节节奏）
- [ ] 风格与半文半白标准一致吗？术语标记规范？

## 七、 常见误区表

### 剧情误区（5项）

| 误区 | 表现 | 解决方案 |
|------|------|----------|
| 战斗过密 | 连续多章高强度战斗 | 在战斗间插入松弛段（盘点/对话/策略） |
| 反转滥用 | 过多反转降低可信度 | 只在关键节点使用反转，每次反转需有伏笔 |
| 线索过多 | 读者跟不上多线发展 | 控制同时推进的线索数量（≤4条） |
| 高潮乏力 | 终局对决不够震撼 | 确保高潮有格局层面的呼应（道统/天命/因果） |
| 收尾仓促 | 高潮后立刻结束 | 给收获盘点和人物反应留足篇幅 |

### 角色误区（7项）

| 误区 | 表现 | 解决方案 |
|------|------|----------|
| 完美人设 | 角色没有缺点 | 给角色真实缺陷并让其付出代价 |
| 被动反应 | 角色只对事件做反应 | 让角色有欲望，欲望驱动行动 |
| 工具化 | 角色只为主角服务 | 给角色独立目标和社交圈 |
| 性格突变 | 行为前后矛盾 | 变化要有铺垫和触发事件 |
| 标签化 | 只有一两个标签 | 通过多面性展示让标签活起来 |
| 过度解释 | 旁白告诉读者角色是什么人 | 用行动和选择展示 |
| 情感廉价 | 动不动流泪感动 | 情感要克制，压抑的情感更有力量 |

### 节奏误区（4项）

| 误区 | 表现 | 解决方案 |
|------|------|----------|
| 开头过慢 | 读者迟迟看不到冲突 | 尽早引入触发事件 |
| 中段疲软 | 对抗阶段缺乏高潮 | 规划关键节点确保每隔几章有波峰 |
| 高潮过短 | 终局对决一笔带过 | 高潮要写足篇幅要有层次感 |
| 收尾过长 | 高潮后拖沓不结尾 | 收获盘点后迅速进入下一阶段伏笔 |

## 八、 规则库引用指引

本文件是方法论总纲，具体的创作法则分布在四个核心规则文件中，按场景自动激活：

| 规则文件 | 激活路径 | 承载内容 |
|---------|---------|---------|
| `core-style.md` | `05_正文/**`、`04_章纲/**` | 文风锚定、展示而非告知、视角纪律、14条禁忌红线 |
| `core-plot.md` | `03_大纲/**`、`04_章纲/**` | 三幕式推进法则、微观场景引擎、六大核心场景类型、多线悬念编织 |
| `core-character.md` | `02_人物/**`、`05_正文/**` | 角色五维法、自主性三铁律、讨喜范式、人性极灰、情感克制 |
| `core-world.md` | `01_世界观/**`、`03_大纲/**` | 冰山涟漪策略、势力生态动力学、道统人格化、命名术语规范 |

**章纲阶段（04_章纲）的特殊说明**：此阶段 `core-style.md` 和 `core-plot.md` 同时激活。`core-plot` 提供结构要求（场景公式、三幕比例），`core-style` 提供语言要求（展示而非告知、术语规范）。结构优先，语言修饰。
