# 知识图谱

> Neo4j 图数据库驱动的小说知识图谱。通过 MCP Server 与 Claude Code 集成，创作过程中自动维护和查询。

---

## 一、图谱 Schema

### 节点类型

| 标签 | 说明 | 核心属性 | 示例 |
|------|------|---------|------|
| `Character` | 角色 | name, realm, faction, status, first_chapter, last_chapter | 江既白 |
| `Faction` | 势力/宗门/家族 | name, level, territory, core_interest, status | 玄天宗 |
| `Location` | 地点 | name, type(城市/秘境/山脉/…), region, features | 寒川渡 |
| `Item` | 物品/法宝/丹药 | name, type, grade, significance | 碧落剑 |
| `Event` | 关键事件 | name, chapter, event_type, outcome, consequences | 寒川渡雪夜 |
| `PowerSystem` | 力量体系/道统 | name, type, characteristics, requirements | 明阳道 |
| `Concept` | 世界观概念 | name, category, description, revelation_level | 灵脉 |
| `Chapter` | 章节 | volume, number, title, status, word_count | 卷1/ch003 |
| `Scene` | 场景 | chapter, sequence, location, time_marker, mood | 卷1/ch003_雪夜对峙 |
| `Secret` | 秘密/信息 | content, secret_type, danger_level | 灵石矿脉真相 |
| `Foreshadowing` | 伏笔 | seed_chapter, foreshadow_type, status, expected_payoff | 主角身世之谜 |

### 关系类型

#### 角色间关系（带时间维度）

| 关系 | 方向 | 属性 | 说明 |
|------|------|------|------|
| `MENTOR_OF` | Character→Character | since_chapter, until_chapter, status(active/ended/turned), trigger | 师徒 |
| `ALLY_OF` | Character→Character | since_chapter, until_chapter, strength, trigger | 盟友 |
| `RIVAL_OF` | Character→Character | since_chapter, intensity, rival_type(竞争/宿敌/暗中对抗) | 对手/宿敌 |
| `KIN_OF` | Character→Character | relation_type(父子/兄妹/夫妻/…), branch | 血缘 |
| `LOVER_OF` | Character→Character | since_chapter, status(暗恋/相恋/决裂/…) | 恋人 |
| `DECEIVED_BY` | Character→Character | since_chapter, what, revealed_chapter | 被欺骗 |
| `SUSPECTS` | Character→Character | since_chapter, what, certainty_level(猜测/怀疑/确信) | 怀疑 |
| `OWES_DEBT_TO` | Character→Character | since_chapter, debt_type(人命/资源/道义), repaid_chapter | 欠债 |
| `PROMISED` | Character→Character | since_chapter, content, fulfilled_chapter | 承诺 |
| `KILLED` | Character→Character | chapter, method, witnesses, consequences | 击杀 |
| `SAVED` | Character→Character | chapter, how, cost | 救命 |

#### 角色 ↔ 势力（带时间维度）

| 关系 | 方向 | 属性 |
|------|------|------|
| `BELONGS_TO` | Character→Faction | since_chapter, role, rank, loyalty_level |
| `BETRAYED` | Character→Faction | chapter, reason, aftermath |
| `FOUNDED` | Character→Faction | chapter, motivation |
| `EXILED_FROM` | Character→Faction | chapter, reason, destination |

#### 角色/势力 ↔ 地点

| 关系 | 方向 | 属性 |
|------|------|------|
| `RESIDES_IN` | Character→Location | since_chapter, reason |
| `BORN_IN` | Character→Location | — |
| `TRAVELS_TO` | Character→Location | chapter, reason, duration |
| `CONTROLS` | Faction→Location | since_chapter, control_type(直属/附庸/影响) |
| `LOCATED_IN` | Location→Location | relation(包含/相邻/通过…连接) |

#### 场景共现

| 关系 | 方向 | 属性 |
|------|------|------|
| `APPEARS_IN` | Character→Scene | role(POV/主动/被动/背景), action_summary |
| `SET_IN` | Scene→Location | — |
| `PART_OF` | Scene→Chapter | sequence |
| `PRECEDES` | Scene→Scene | transition(时间跳跃/空间切换/平行剪辑) |
| `TRIGGERS` | Scene→Scene | causality(直接导致/间接引发/暗线关联) |

#### 信息差追踪

| 关系 | 方向 | 属性 |
|------|------|------|
| `KNOWS` | Character→Secret | since_chapter, source(亲眼/被告知/推测/窃听) |
| `HIDES_FROM` | Character→Secret | target_characters, since_chapter, why |
| `DISCOVERS` | Character→Secret | chapter, how, reaction |
| `MISUNDERSTANDS` | Character→Secret | what_they_think, reality, chapter, corrected_chapter |
| `RELATES_TO` | Secret→Character | connection_type |
| `RELATES_TO` | Secret→Event | connection_type |
| `RELATES_TO` | Secret→Item | connection_type |

#### 伏笔链路

| 关系 | 方向 | 属性 |
|------|------|------|
| `PLANTED_IN` | Foreshadowing→Chapter | seed_type(对话/细节/事件/象征) |
| `DEVELOPS_IN` | Foreshadowing→Chapter | progress_note |
| `RESOLVED_IN` | Foreshadowing→Chapter | payoff_quality(惊艳/满意/平淡/遗漏) |
| `INVOLVES` | Foreshadowing→Character | role(埋设者/回收者/被影响者) |
| `ECHOES` | Foreshadowing→Foreshadowing | echo_type(呼应/对比/升级/反转) |
| `SETUP_FOR` | Foreshadowing→Event | — |

#### 其他

| 关系 | 方向 | 属性 |
|------|------|------|
| `POSSESSES` | Character→Item | since_chapter, how_obtained, lost_chapter |
| `PRACTICES` | Character→PowerSystem | since_chapter, mastery_level |
| `OCCURRED_AT` | Event→Location | — |
| `PARTICIPATED_IN` | Character→Event | role(发起/参与/受害/旁观) |
| `ALLIED_WITH` | Faction→Faction | since_chapter, alliance_type |
| `ENEMY_OF` | Faction→Faction | since_chapter, conflict_nature |
| `VASSAL_OF` | Faction→Faction | since_chapter, terms |

## 二、MCP 工具使用指南

### 可用工具

**neo4j-memory**（CRUD 操作）：
- `create_entities` — 批量创建节点
- `create_relations` — 批量创建关系
- `add_observations` — 为已有节点添加观察信息
- `search_nodes` — 按名称搜索节点
- `open_nodes` — 按名称打开节点详情
- `read_graph` — 读取整个图谱
- `delete_entities` / `delete_relations` / `delete_observations` — 删除操作

**neo4j-cypher**（高级查询）：
- `get-schema` — 获取当前图谱结构
- `read-query` — 执行只读 Cypher 查询
- `write-query` — 执行写操作 Cypher 查询

### 常用查询模板

```cypher
-- 查某角色的完整关系网
MATCH (c:Character {name: '江既白'})-[r]-(n) RETURN c, r, n

-- 查某章所有出场角色
MATCH (c:Character)-[:APPEARS_IN]->(s:Scene)-[:PART_OF]->(ch:Chapter {volume: 1, number: 3})
RETURN c.name, s

-- 查未回收的伏笔
MATCH (f:Foreshadowing {status: 'active'}) RETURN f

-- 查某角色知道哪些秘密
MATCH (c:Character {name: '江既白'})-[:KNOWS]->(s:Secret) RETURN s.content

-- 查两个势力间的关系路径
MATCH path = shortestPath((f1:Faction {name: '玄天宗'})-[*..5]-(f2:Faction {name: '幽冥宗'}))
RETURN path

-- 查信息差：谁被蒙在鼓里
MATCH (c:Character)-[:MISUNDERSTANDS]->(s:Secret) RETURN c.name, s.content, s.what_they_think
```

## 三、更新规则

### 何时更新图谱

| 触发场景 | 操作 |
|---------|------|
| 创建新角色 (`/create-character`) | 创建 Character 节点 + 所有关键关系 |
| 完成章节 (`/write-chapter`) | 创建 Scene 节点 + APPEARS_IN + 新 Event/Secret |
| 设计世界观 (`/design-world`) | 创建 Location/Faction/PowerSystem/Concept 节点 |
| 设计大纲/细纲 | 创建 Event 节点 + 伏笔 PLANTED_IN |
| 伏笔回收 | 更新 Foreshadowing 状态 + RESOLVED_IN |
| 角色关系变化 | 更新关系属性 since_chapter/status/trigger |
| 审阅章节 (`/review-chapter`) | 补充遗漏的 APPEARS_IN / Scene 共现 |

### 谁负责更新

- **Claude 自动更新**：在 `/write-chapter`、`/create-character`、`/design-world` 等 skill 执行过程中，Claude 自动调用 MCP 工具同步图谱
- **手动批量同步**：使用 `/kg-update` skill 扫描文件批量更新
- **人类审核确认**：重要关系变更由人类确认后再写入

## 五、命名规范

| 规则 | 说明 |
|------|------|
| Character.name | 与文件名一致：`前缀_姓名`（如 `主角_江既白`） |
| Faction.name | 中文全称（如 `玄天宗`） |
| Location.name | 中文地名（如 `寒川渡`） |
| Event.name | 简短描述（如 `寒川渡雪夜对峙`） |
| Foreshadowing.name | 伏笔简述（如 `主角身世之谜`） |
| Secret.content | 简短描述（如 `灵石矿脉实为封印`） |
| Scene.name | `卷X/chNNN_场景简称`（如 `卷1/ch003_雪夜对峙`） |
