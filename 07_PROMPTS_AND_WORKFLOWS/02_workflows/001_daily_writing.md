# 日常写作流程

> 每日创作工作流程，基于写作要求.md的风格标准

---

## 每日启动流程（15分钟）

### 1. 读取进度
```
读取 00_PROJECT_FOUNDATION/PROGRESS.md
```
了解当前进度和目标。

### 2. 读取当前任务
```
读取 00_PROJECT_FOUNDATION/TASKS.md
```
确定今天的具体写作目标。

### 3. 加载上下文
**必须读取**：
- 当前章节的章纲
- 当前出场人物的SUMMARY.md
- 当前场景涉及的世界观设定

**按需读取**：
- 前一章的正文（用于衔接）
- 相关伏笔记录

---

## 创作流程（2-4小时）

### 步骤1：确认章节目标
从章纲中确认本章的：
- 核心叙事功能（推动情节/揭示信息/展现代价/深化主题）
- 三幕结构（钩子→博弈→收束）
- 出场人物和场景设定

### 步骤2：检查风格标准
写作前快速回顾 `00_PROJECT_FOUNDATION/STYLE_GUIDE/写作要求.md` 中的：
- 12条禁止规则
- 段落公式：[景-人-心]三段式
- 内心独白规范（『』包裹）

### 步骤3：写作正文
严格遵循写作要求.md中的风格规范：

**开场**（20%篇幅）：
- 3句话内建立场景、绑定视角、抛出本章核心悬念
- 先用环境渲染（阴沉/肃杀/宏大/幽静/诡谲）

**发展**（60%篇幅）：
- 通过对话、行动、视角切换推进核心事件
- 高频使用内心独白（『』包裹）
- 遵循[景-人-心]段落公式
- 实现人物状态实质性变化

**收束**（20%篇幅）：
- 给出阶段性结果
- 埋下新伏笔
- 结尾句需有余韵

### 步骤4：自检清单
完成章节后，逐项检查：

- [ ] 无直接情绪告知（"他很愤怒"类表述）
- [ ] 无作者评论或旁白说明
- [ ] 场景内视角固定（无上帝视角切换）
- [ ] 对话有信息推进（无闲聊）
- [ ] 内心独白使用『』包裹
- [ ] 遵循[景-人-心]段落结构
- [ ] 专用术语使用『』或【】包裹
- [ ] 半文半白语言风格
- [ ] 字数达标（约6000字）

---

## 结束流程（30分钟）

### 1. 更新章节元数据
```
更新 04_MANUSCRIPT/volume_XX/XXX_chapter_YY/metadata.md
```

### 2. 更新追踪表
- 更新人物出场追踪表（`02_CHARACTER_DEVELOPMENT/00_appearance_tracker.md`）
- 更新剧情线索追踪（如涉及主线/支线推进）
- 更新伏笔追踪表（如设置或回收了伏笔）

### 3. 更新进度
```
更新 00_PROJECT_FOUNDATION/PROGRESS.md
```

### 4. 记录灵感
如有新灵感或需要调整的内容，记录到：
```
05_SCENES_AND_NOTES/02_inspiration/
```

### 5. Git提交
```bash
git add 04_MANUSCRIPT/volume_XX/XXX_chapter_YY/
git commit -m "feat(volX/chYYY): 完成第YYY章初稿（XXXX字）"
```

---

## 质量标准速查

### 必须做到
- 展示而非告知（Show, Don't Tell）
- 内心独白驱动叙事
- [景-人-心]段落结构
- 信息密度最大化
- 因果链严密

### 绝对禁止
- 直接告知情绪
- 作者评论/说教
- 视角混乱
- 术语矛盾
- 对话闲聊
- 儿戏战斗
- 人物降智
- 冗长拖沓
