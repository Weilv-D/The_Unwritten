# 提交格式
# <type>(scope): 中文摘要
#
# type 仅允许以下几类：
# feat     新增内容、设定、能力或脚本
# fix      修正错误、一致性问题或错误表述
# refactor 重构结构与组织方式，不改变既定目标
# docs     文档、索引、导航、模板与说明更新
# chore    配置、清理、维护类杂项
#
# scope 规则：
# 1. 优先使用中文模块名，例如：世界观 / 大纲 / 卷一 / 方法论 / 提交
# 2. 只有在必须精确指代时，才使用文件名、工具名或模块名
# 3. scope 可省略，但不建议长期省略
#
# 摘要规则：
# 1. 使用中文动宾短句，首词优先：新增 / 完善 / 修正 / 重构 / 整理 / 移除 / 更新
# 2. 一次提交只表达一个主题，避免把无关事项揉进同一条提交
# 3. 不写空话，不使用 feature X / update docs / fix bug 之类占位描述
# 4. 不写句号，不堆砌三个以上并列动作
#
# 推荐示例：
# feat(世界观): 新增天地业位体系
# fix(卷一): 修正章纲时间线冲突
# refactor(方法论): 重组创作流程结构
# docs(提交): 新增中文提交模板
# chore(脚本): 清理过时检查脚本
